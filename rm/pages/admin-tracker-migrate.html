Activity Ledger → Tracker Migration

What this does

One-shot migration of your userData/{uid}/activityLedger rows into trackerEntries/{uid}/entries (the lab-apps activity tracker). After this, the tracker becomes the single source of truth for daily activity.

Mapping summary: RM categories → lab-apps tree per the table reviewed in chat. Skipped: noise and personal top-level categories. Uncategorized: rows with category=unknown.

Idempotency: each migrated source row is stamped with migrated_to_tracker_id. Re-running this page skips rows that already have that field, so it's safe to retry after a partial run.
