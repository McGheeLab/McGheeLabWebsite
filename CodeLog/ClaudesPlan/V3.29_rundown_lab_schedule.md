# V3.29 — Huddle Rundown + Lab Schedule

## Goal

Add two interconnected features to the Huddle app:
1. **The Rundown** — Weekly non-timeframe-based task lists where students post what they plan to do, others can request to join/shadow, and the task owner schedules it on the calendar
2. **Lab Schedule / Availability** — Users define their recurring weekly availability and unavailability (rigid vs flexible), enabling schedule overlap detection when coordinating Rundown tasks

## Approach

### Data Model
- **`huddleRundown`** — Weekly task items with category, project association, join requests array, status lifecycle (open → scheduled → done)
- **`huddleConfig/settings`** — Lab-wide admin-configurable categories for rundown tasks (follows meetingConfig pattern)
- **`huddleScheduleTemplates/{userId}`** — Recurring weekly availability template with blocks per day-of-week
- **`huddleScheduleOverrides`** — Per-week deviations from template (add/remove/replace actions)

### Navigation
5 new sidebar tabs added:
- **Feeds**: Rundown, Team Availability
- **Create**: Add Task
- **My Stuff**: My Tasks, My Schedule

### Key Decisions
- Rundown tasks are a **separate collection** from huddlePlans — different lifecycle (unscheduled → become plans when scheduled)
- Schedule uses **recurring template + per-week overrides** so users don't re-enter every week
- Admin-configurable categories stored in `huddleConfig/settings` (admin write, auth read)
- Unavailability blocks have **rigid/flexible** distinction with visual encoding (hatched vs dashed)

## Files Modified
- `apps/huddle/app.js` — ~800 lines added: state, CRUD, subscriptions, 5 render/wire functions, 4 modals
- `apps/huddle/styles.css` — ~300 lines added: rundown cards, schedule grid, team Gantt view, category manager
- `firestore.rules` — 4 new collection rule blocks

## User Flows
1. **Add Task**: Create → describe task → pick category + project → appears in Rundown feed
2. **Request to Join**: Browse Rundown → click "Request to Join" → select skill level + note
3. **Accept & Schedule**: Owner sees pending request → Accept → Schedule modal with availability overlap → Creates huddlePlan
4. **Set Availability**: My Schedule → drag to create blocks → choose available/unavailable, reason, rigidity
5. **View Team**: Team Availability → select day → see Gantt bars for all members → click to compare overlap
