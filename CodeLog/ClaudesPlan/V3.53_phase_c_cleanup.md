# V3.53 — Phase C cleanup: delete `/apps/`, retire `lab-apps.js` load, migrate deep-links

## Goal

Close out the lab-apps-into-RM migration. After Phase B (V3.41–V3.52)
every lab app at `/apps/<name>/` had either a native RM page that
superseded it, or had been deleted as a stub. V3.53 deletes those
now-redundant `/apps/` directories outright, removes the orphaned
`lab-apps.js` script load from the public site, drops `APPS_CACHE`
from the service worker, and migrates every remaining `/#/apps/<name>`
URL reference (manifest.json shortcuts, Cloud Function push
notifications) to the corresponding `/rm/pages/<name>.html` native
page.

## What's preserved (per user guidance)

- **`lab-apps.js`** stays in the repo. The user wants the
  `_splitAppId` / `_splitRatio` split-view harness (originally built
  to show two iframe-embedded lab apps side-by-side) as a future
  reference for a general RM split-view feature. The file is no
  longer loaded by `index.html`, so the dead lab-app-iframe code
  doesn't run anywhere — but the source stays available. A future
  phase will lift the split-view UX into RM proper.
- **Public-site SPA `#/dashboard`, `#/admin`, `#/cv`, `#/guide`
  routes** stay intact. The user mentioned future workflow-driven
  page restructuring; these routes remain operational until that
  effort scopes them. V3.53 deliberately doesn't touch
  `user-system.js`, `cv-builder.js`, or the `app.js` SPA router
  beyond the cleanup directly required by the `/apps/` deletion.

### Iframe-bridge wrapper redirects

`git rm rm/pages/app-*.html` — five files:

- `rm/pages/app-chat.html`
- `rm/pages/app-equipment.html`
- `rm/pages/app-huddle.html`
- `rm/pages/app-meetings.html`
- `rm/pages/app-scheduler.html`

Each was already a `<meta http-equiv="refresh">` redirect to its
native RM target (replaced when each app's V3.4x port flipped the
nav). Bookmarks pinned to these URLs from Phase A return a 404
after V3.53 — acceptable because the same URLs were always one
hop short of the native page anyway. The `nav.js` Operations
group already points at the native pages (V3.41–V3.52 per-port
flips); no nav changes needed in V3.53.

## Changes that need a Firebase Functions deploy

Cloud Functions generate push-notification deep-links. The lab-app
era used `/#/apps/<name>` URLs; with `/apps/` gone, those resolve
to dead routes. Migrated to `/rm/pages/<name>.html`:

| File | Lines | Old URL | New URL |
|---|---|---|---|
| `functions/equipment.js` | 35, 62, 72 | `/#/apps/equipment` | `/rm/pages/equipment.html` |
| `functions/huddle.js` | 44, 60, 80, 104, 136 | `/#/apps/huddle` | `/rm/pages/huddle.html` |
| `functions/chat.js` | 49 | `/#/apps/chat?channel=${channelId}` | `/rm/pages/chat.html?channel=${channelId}` |
| `functions/test-notifications.js` | 18, 24, 30 | `/#/apps/chat?channel=...` | `/rm/pages/chat.html?channel=...` |
| `functions/test-notifications.js` | 36, 42, 48, 54 | `/#/apps/huddle` | `/rm/pages/huddle.html` |
| `functions/test-notifications.js` | 60, 66, 72 | `/#/apps/equipment` | `/rm/pages/equipment.html` |
| `functions/calendar-proxy.js` | 7 (comment) | `apps/shared/calendar-service.js` | `/rm/js/calendar-service.js` |

After merge:

```bash
firebase deploy --only functions
```

Without this, push notifications routed to users on equipment /
huddle / chat will deep-link into 404 pages.

## Other changes

- **`manifest.json`** — `start_url` changed from `/#/apps` →
  `/rm/`. Four `shortcuts[].url` entries updated from
  `/#/apps/<name>` to `/rm/pages/<name>.html`. PWA users with the
  app installed on their home screen will re-resolve the shortcuts
  on next install / reinstall.
- **`index.html`** — `<script defer src="lab-apps.js">` removed.
  Comment in place explaining the file is preserved for the
  split-view harness reference.
- **`sw.js`** — `APPS_CACHE` constant + `ALL_CACHES` entry +
  runtime `staleWhileRevalidate` `/apps/*` routing all removed.
  `/lab-apps.js` dropped from `SHELL_URLS`. `CACHE_VERSION` bumped
  27 → 28.

## Known follow-ups (not blockers)

- **Settings Calendar tab** — the V3.48 port skipped the lab-app
  Calendar Integration UI. With CalendarService now in RM (V3.51),
  a future phase can lift the calendar UI as a third tab next to
  Profile + Notifications.
- **Scheduler My Schedule tab** — the V3.47 port covered My
  Schedulers only. With ScheduleService + CalendarService in RM
  (V3.51), a future phase can lift the My Schedule personal
  calendar UI.
- **Split-view harness** — preserve and adapt the `_splitAppId` /
  `_splitRatio` code from `lab-apps.js` into a general RM
  split-view feature so users can keep two pages open
  side-by-side (e.g. Lab Chat + Activity Tracker, Procurement +
  Inventory). User flagged this as a wanted feature.
- **Workflow-driven page restructuring** — the user mentioned
  future restructuring to align RM pages with student workflows
  at different levels. Public-site `#/dashboard`, `#/admin`, etc.
  routes may be re-homed during that effort.
- **Firestore-backed OAuth tokens** — currently CalendarService
  stores Google / Outlook tokens in `sessionStorage`. Moving to
  `oauthConnections/{uid}` Firestore docs would let admins skip
  re-OAuth on a fresh browser session. Pure enhancement, no
  rush.
