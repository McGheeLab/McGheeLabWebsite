# V2.2 — Project Packages, Opportunities Page, Dark Theme

## Goals
1. **Project packages** — Let grad/postdoc/PI compile published stories into projects with outcomes
2. **Opportunities page** — Replace Contact with a job board managed by admins
3. **Default role by category** — Auto-set editor for grad/postdoc, contributor for undergrad/highschool
4. **Dark theme consistency** — Restyle all user system pages to match the main site

## Key Decisions

### Project Packages
- Separate Firestore collection `projectPackages` (not mixed with stories)
- Stories are selected by checkbox from all published stories
- Outcomes text is required before publishing — enforced in `saveProject()`
- Only users with `canCreateProject()` (admin, or category in `PROJECT_CATEGORIES`) see the project editor
- Dashboard conditionally renders "My Projects" card via `Auth.canCreateProject()`

### Default Roles
- `CATEGORY_DEFAULT_ROLE` mapping: `{ postdoc: 'editor', grad: 'editor', undergrad: 'contributor', highschool: 'contributor' }`
- `syncRoleToCategory()` runs on category change AND profile selection in the invitation form
- Admin can still manually override the role if needed

### Opportunities
- Simple CRUD: admin creates via form in admin panel, public reads on `#/opportunities`
- Fields: title, type (dropdown), description, requirements, deadline, contact email, status (open/closed)
- Firestore rule: public read, admin write
- Old `#/contact` route redirects to `#/opportunities` for backward compat

### Dark Theme
- Complete rewrite of `user-styles.css` (~700 lines)
- Uses CSS variables from main site: `--bg`, `--surface`, `--surface-2`, `--text`, `--muted`, `--accent`, `--link`, `--radius`, `--shadow`
- Semi-transparent backgrounds (`rgba(255,255,255,.04)`) instead of solid whites
- Status badges use transparent colored backgrounds with lighter text
- All interactive elements (buttons, inputs, tabs, upload zones) match the main site's aesthetic
