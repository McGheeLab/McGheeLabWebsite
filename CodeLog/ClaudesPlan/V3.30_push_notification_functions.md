# V3.30 — Firebase Cloud Functions for Push Notifications

## Goal
Complete the push notification pipeline by adding server-side Firebase Cloud Functions that send FCM messages when Firestore events occur. The client-side infrastructure (token registration, service worker, settings UI) already exists — this adds the missing "send" side.

## Approach
- Create a `functions/` directory with Firebase Cloud Functions v2 (2nd gen)
- Use Firestore triggers (onCreate, onUpdate) to detect notification-worthy events
- Core helper (`helpers/notify.js`) handles: user settings checks, token lookup, FCM send, stale token cleanup
- Quiet hours are enforced client-side by the existing service worker — functions just send

## Files Created
1. `firebase.json` — Firebase deployment config (functions + Firestore rules)
2. `functions/package.json` — Node.js project (firebase-admin, firebase-functions)
3. `functions/index.js` — Main entry, exports all 6 Cloud Functions
4. `functions/helpers/notify.js` — Core: build FCM payload, send to users, clean up stale tokens
5. `functions/helpers/users.js` — `sendToAllMembers()` for lab-wide notifications
6. `functions/chat.js` — `onChatMessageCreate`: notifies channel subscribers on new message
7. `functions/huddle.js` — `onHuddlePlanUpdate`, `onHelpRequestCreate`, `onHelpRequestUpdate`
8. `functions/equipment.js` — `onEquipmentBookingCreate`, `onEquipmentBookingUpdate`

## Functions Summary

| Function | Trigger | Who Gets Notified |
|---|---|---|
| `onChatMessageCreate` | `chatMessages/{id}` create | Channel subscribers (respects muted, mentionsOnly) |
| `onHuddlePlanUpdate` | `huddlePlans/{id}` update | Owner (on join/watch), joiners+watchers (on status change) |
| `onHelpRequestCreate` | `huddleHelpRequests/{id}` create | All lab members |
| `onHelpRequestUpdate` | `huddleHelpRequests/{id}` update | Request owner (on new response) |
| `onEquipmentBookingCreate` | `equipmentBookings/{id}` create | Equipment managers (if pending-approval) |
| `onEquipmentBookingUpdate` | `equipmentBookings/{id}` update | Booker (on confirmed/displaced) |

## Key Decisions
- **Cloud Functions v2** — better auto-scaling, simpler API
- **No client-side changes** — existing infrastructure is complete
- **Settings-aware** — respects `userSettings/{uid}.notifications.enabled` and per-app toggles
- **Token cleanup** — removes stale FCM tokens on send failure
- **Notification grouping** — uses `tag` field to group notifications per-channel/per-event

## Deployment
```bash
cd functions && npm install
firebase deploy --only functions
```
