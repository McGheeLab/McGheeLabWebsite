# V3.26 — Settings App & Lab Apps Bottom Nav Refactor

## Goal
Add a unified Settings app to the lab apps ecosystem, accessible from the bottom nav bar. Make the bottom nav icon-only on mobile for a cleaner phone experience.

## Approach
1. Add Settings entry to the `LAB_APPS` registry — automatically appears in hub grid and bottom nav
2. Build `apps/settings/` as a standalone app (same iframe pattern as all other lab apps)
3. Hide text labels in the lab apps bottom nav on mobile (<=700px)
4. Add Firestore collection `userSettings/{uid}` for notification preferences and quiet hours
5. Integrate quiet hours into the push notification service worker

## Key Decisions
- Settings is a regular lab app (iframe), not a special SPA route — consistent with all other apps
- Per-app admin detection reads `meetingConfig/settings.meetingAdmins[]` and `chatConfig/settings.chatAdmins[]` from Firestore
- Admin settings link to existing app settings (no UI duplication) via parent navigation
- Quiet hours cached in IndexedDB (`mcgheelab-prefs`) so the service worker can check without network
- Global admins see all admin sections + site admin links

## Files Created
- `apps/settings/index.html` — Settings app HTML shell
- `apps/settings/app.js` — Profile editing, notification prefs, admin detection, quiet hours sync
- `apps/settings/styles.css` — Toggle switches, form fields, admin cards, section layout

## Files Modified
- `lab-apps.js` — Added Settings entry to `LAB_APPS` array (gear icon, active, non-admin-only)
- `user-styles.css` — `.lab-app-bottom-label { display: none }` at <=700px for icon-only mobile nav
- `firestore.rules` — Added `userSettings/{userId}` rules (owner read/write, admin read)
- `firebase-messaging-sw.js` — Added quiet hours check: reads from IndexedDB, suppresses notification during quiet hours (badge still increments)
- `sw.js` — Added settings app files to precache list
