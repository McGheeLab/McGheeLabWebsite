# V3.8 — Session Builder: Calendar + Drag-to-Select Time Grid

## Goal
Replace the current manual session-time entry (type start/end times) with a visual builder: select days on a calendar → drag a time window on a scrollable time grid → that window applies to all selected days. Repeat for additional sessions. Previous sessions shown in a different color on the time grid. Calendar days colored to reflect which session groups they belong to.

## How It Works

1. **Calendar multi-select** — At the bottom of the dashboard scheduler creation area. Click days to select them (highlighted in an "active selection" color).
2. **Time grid appears** — When 1+ days are selected, a scrollable vertical time table shows (7am–9pm, 15-min rows). Previously saved sessions shown as colored blocks.
3. **Drag to select time range** — User clicks and drags vertically on the time grid to highlight a time window. This window applies to ALL currently selected days.
4. **Apply** — Releasing the drag (or clicking an Apply button) saves the session block. Calendar days update to show a "saved" color matching that session's color.
5. **Add more** — User can now select different days and drag another time window. Each session group gets a distinct color.
6. **Edit/remove** — Click a saved session block on the time grid to remove it.

## Data Model
```javascript
// Each session block: a time window applied to specific days
// Stored on the schedule document
sessionBlocks: [
  { id: 'sb-1', days: ['2026-06-08', '2026-06-10'], start: '09:00', end: '10:30', color: 0 },
  { id: 'sb-2', days: ['2026-06-09', '2026-06-11'], start: '14:00', end: '15:30', color: 1 },
]

// color is an index into a palette array, not an actual hex value
// Backward compat: expandSchedule() can derive slotDefs + selectedDays from sessionBlocks
```

## Backward Compatibility
`expandSchedule()` updated:
- If `sessionBlocks` present → derive days and slots from them
- Else fall back to `selectedDays` + `slotDefs` → then `startDate/endDate`

## Session vs Freeform
- **Sessions (this builder)** — Use when external constraints define specific time windows on specific days. Visual drag builder.
- **Freeform** — Use when you want guests to paint their own availability across a uniform grid. Keep existing calendar + hour range + granularity setup.

## UI Components

### Calendar (enhanced)
- Existing `calendarHTML()` / `wireCalendar()` enhanced with:
  - Day cells show colored dots/backgrounds for each session group they belong to
  - Active selection (days being picked now) shown in a highlight color distinct from saved session colors

### Time Grid
- Scrollable vertical column, 7am–9pm, 15-min rows
- Each row: time label on left, selectable area on right
- Saved session blocks rendered as colored overlays
- Drag interaction: mousedown → mousemove highlights rows → mouseup saves
- Click on existing block to delete it

### Session List
- Below the time grid: summary of all session blocks
- Each shows: days, time range, color swatch, delete button
