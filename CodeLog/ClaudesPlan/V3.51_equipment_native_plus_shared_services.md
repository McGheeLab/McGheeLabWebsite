# V3.51 — Equipment native port + shared services into RM

## Goal

Two deliverables in one phase:

1. **Equipment native port** — replace the V3.40 iframe wrapper at
   [rm/pages/app-equipment.html](../../rm/pages/app-equipment.html)
   with a native renderer at [rm/pages/equipment.html](../../rm/pages/equipment.html).
   Same lift-and-patch playbook used for V3.41 / V3.49 / V3.50 / V3.52.
2. **Shared services into RM** — copy CalendarService,
   ScheduleService, and ScheduleUtils from `/apps/shared/` into
   `/rm/js/` and load them on the three deferred-feature pages
   (huddle, activity-tracker, scheduler). The deferred features
   (Lab Schedule view, calendar-import panel, future My Schedule
   tab on scheduler) light up in production.

## Re-scope from the original V3.51 plan

The V3.40 plan scoped V3.51 as "equipment + sessionStorage→Firestore
OAuth token refactor". On execution it became clear that the token
refactor is an **enhancement**, not a blocker:

- The standalone lab apps work fine with sessionStorage tokens —
  re-OAuth once per browser session is the existing UX.
- Bringing the services into RM doesn't change that — sessionStorage
  is per-frame, but RM is a single same-origin page, so the token
  survives across page navigations within RM exactly as it does in
  the standalone lab app.
- The Firestore-backed token refactor would let admins skip the
  re-OAuth step on a fresh browser session, but isn't required for
  the deferred features to work.

So V3.51 ports equipment, brings the services in verbatim, and
lights up the deferred features. The token-storage refactor is now
**V3.53** scope (or later — purely an enhancement).

## Equipment port

### Patches applied to the lifted file

1. **`window.McgheeLab = window.McgheeLab || {}`** at top.
2. **Container id `#app` → `#equipment-root`**.
3. **`db()` prefers `firebridge.db()`** with `firebase.firestore()`
   fallback.
4. **Bootstrap rewrite (lab `app.js` lines 145–195)** — replaces
   the dual AppBridge / direct-Firebase path with the RM contract:
   `firebridge.gateSignedIn()` → `await firebridge.whenAuthResolved()`
   → `_user/_profile` from firebridge → `Promise.all([loadSettings,
   loadTraining, loadUsers])` → `render()` → `subscribeEquipment()`
   → `subscribeUsers()`. `MobileShell.enableTabSwipe` stays guarded.
   `firebridge.onAuth` re-renders on role flips so newly-promoted
   admins see the Admin tab without a manual reload.
5. **`notifyResize()` AppBridge guard** — patched from
   `if (!McgheeLab.AppBridge.isEmbedded()) return;` to also
   null-check `AppBridge` itself + `.isEmbedded` so it no-ops cleanly.

### Google Calendar admin push-sync

The lab app's "Push bookings to Google Calendar" feature uses Google
Identity Services (`google.accounts.oauth2.initTokenClient`) with the
token stored in `sessionStorage['equip_gcal_token']`. The renderer
code transfers verbatim — only the host page's script tags change:
`<script src="https://accounts.google.com/gsi/client">` is loaded by
[rm/pages/equipment.html](../../rm/pages/equipment.html).

Token storage stays in sessionStorage for V3.51. Admins re-OAuth
once per browser session, same as the standalone lab app.

## Key decisions

- **Token storage stays in sessionStorage.** The "OAuth refactor"
  framing in the V3.40 plan turned out to conflate two independent
  things — bringing services into RM vs changing token storage.
  V3.51 does the former (necessary to light up deferred features);
  the latter is a pure enhancement that can ship independently
  whenever appetite allows.
- **Verbatim service copies, no patches.** CalendarService /
  ScheduleService / ScheduleUtils self-attach to
  `window.McgheeLab.X` and use `firebase.firestore()` directly —
  same semantics in RM as in the standalone lab apps. Phase C
  deletes `/apps/shared/`; the RM copies become canonical.
- **No deletion of `/apps/equipment/` in V3.51.** Standalone URL
  still resolves as a fallback. V3.53 Phase C cleanup deletes
  every remaining `/apps/` directory + the `/apps/shared/` services
  + `lab-apps.js`.
- **Settings Calendar tab deferred to V3.53.** Bigger UI lift than
  fits cleanly here; lighting up the three other deferred features
  via just-script-tag wiring is much smaller scope. V3.53 ports
  the calendar UI properly.
