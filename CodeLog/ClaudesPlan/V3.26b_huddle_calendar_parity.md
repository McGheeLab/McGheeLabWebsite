# V3.26b — Huddle Calendar Parity + Phone Scaling

## Goal
Port all equipment scheduler calendar features (freeze-pane, zoom, slider, phone scaling) to the Huddle app's time grid views.

## Approach
Mirror the equipment scheduler's calendar architecture into the huddle app:
- Same ZOOM_LEVELS array and zoom state pattern
- Same freeze-pane flex layout (cal-header fixed, cal-body scrollable)
- Same custom time slider (proportional thumb, auto-hide)
- Same pinch-to-zoom gesture handler
- Sticky day headers within the scroll container
- Phone-width scaling so the 7-day grid fits without horizontal scroll

## Files Modified
- `apps/huddle/app.js` — zoom state, buildTimeGrid zoom-aware rows, freeze-pane HTML structure for Plan Feed + My Plans (daily/weekly), wireCalFeatures/wireZoom/wireTimeSlider/wirePinchZoom functions, sticky head classes on headers
- `apps/huddle/styles.css` — hud-cal-layout flex column, hud-cal-header/body, hud-grid-wrap scroll container with hidden scrollbar, hud-time-slider, hud-zoom-controls, hud-legend-row, hud-sticky-head, mobile grid scaling overrides

## Key Decisions
1. **Shared pattern, not shared code**: huddle and equipment are independent apps — copied the pattern rather than extracting a shared module, keeping each self-contained
2. **View toggle in cal-header**: for day/week views the view toggle is inside the fixed header so it doesn't scroll away; for month view it stays standalone
3. **Phone grid scaling**: `min-width: 0` + `grid-template-columns: 36px repeat(7, 1fr)` on mobile forces the grid to fit phone width; font sizes scaled down proportionally
4. **`:has()` for overflow control**: `.hud-main:has(.hud-cal-layout)` switches to `overflow: hidden; padding: 0` only when a calendar view is active
