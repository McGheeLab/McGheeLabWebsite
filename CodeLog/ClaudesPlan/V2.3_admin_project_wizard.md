# V2.3 — Admin Project Wizard

## Goal
Refine the project system: admin-only creation, step-by-step wizard, ordered story assignment with delete, external links, and future classes editing prep.

## Key Decisions

### Admin-Only Projects
- `canCreateProject()` simplified to `Auth.isAdmin()` — removed `PROJECT_CATEGORIES`
- Only PI-level users see the "Project Packages" section on the dashboard
- Grad students and postdocs still get editor role (for story auto-publish) but cannot create projects

### 5-Step Wizard
1. **Details** — Title, description, outcomes (required for publish), external link (optional)
2. **Team** — Author (auto-filled PI), mentor dropdown, contributor multi-select with chips
3. **Stories** — Add from dropdown of published stories, ordered list with up/down/remove controls
4. **References** — Same ref blocks as before (type, title, URL, detail)
5. **Review** — Summary of all fields; warns if outcomes are missing; Publish button appears here

Navigation: step indicators (click to jump), Previous/Next buttons, Save Draft on every step.

### Ordered Story Assignment
- Stories stored as ordered array `storyIds` (not a Set)
- Admin selects from dropdown (filtered to exclude already-assigned), clicks Add
- Each story row shows: order number, title, author, up/down/remove buttons
- Dropdown refreshes after each add/remove to prevent duplicates

### External Links
- New `link` field on project data
- Rendered as "View Project Site" button on public Projects page
- Optional — no validation beyond URL type input

## Files Modified
- `user-system.js` — Rewrote `renderProjectEditor` (wizard HTML) and `wireProjectEditor` (wizard logic, story ordering)
- `user-styles.css` — Added wizard steps, panels, nav, story row, review summary styles
- `app.js` — Added external link rendering for Firestore projects
- `firestore.rules` — No changes needed (projectPackages rules already cover this)
- `CodeLog/Updates/CHANGELOG.md` — V2.3 entry
