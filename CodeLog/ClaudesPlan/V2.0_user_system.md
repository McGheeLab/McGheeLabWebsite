# V2.0 — User System & Story Editor

**Date:** 2026-03-20
**Goal:** Add user accounts, invitation-based registration, profile editing, story editor, and admin panel to the McGheeLab site using Firebase.

## Architecture

```
Public Site (existing)          Firebase Backend
 index.html / app.js  ←------→  Auth (email/password)
 content.json (fallback)         Firestore (users, stories, invitations)
                                 Storage (images at 3 resolutions)

New Routes:
 #/login              Login / Register (invitation-gated)
 #/dashboard          Profile editor + story list
 #/dashboard/story/:id Story editor (create/edit)
 #/admin              User management, invitations, story review
```

## New Files

| File | Purpose |
|------|---------|
| `firebase-config.js` | Firebase initialization (user fills in project credentials) |
| `user-system.js` | All user system logic: auth, DB ops, dashboard, story editor, admin |
| `user-styles.css` | Styling for dashboard, admin, story editor, forms |
| `firestore.rules` | Firestore security rules |
| `storage.rules` | Storage security rules |
| `migrate-content.js` | One-time migration of content.json to Firestore |

## Modified Files

| File | Changes |
|------|---------|
| `index.html` | Firebase SDK tags, new CSS/JS refs, Dashboard/Admin/Login nav links |
| `app.js` | Router extended for new routes, Firestore story integration on projects page |

## Data Model (Firestore)

- **users/{uid}** — name, email, role (admin/editor/contributor), category, photo URLs, bio
- **stories/{id}** — title, project, description, authorUid, sections[], status (draft/pending/published)
- **invitations/{id}** — email, role, category, expiresAt, used, createdBy

## Privilege System

| Role | Can Edit Profile | Can Create Stories | Auto-Publish | Can Manage Users |
|------|-----|-----|-----|-----|
| admin | Yes | Yes | Yes | Yes |
| editor | Yes | Yes | Yes | No |
| contributor | Yes | Yes | No (pending review) | No |

## Key Decisions

1. **Firebase over PHP/MySQL** — decoupled from GoDaddy hosting, already partially used in poster/
2. **Compat SDK** — works with vanilla JS script tags, no build step needed
3. **Client-side image resize** — Canvas API generates thumb (300px), medium (800px), full (1600px) as webp
4. **Hybrid data** — content.json renders immediately, Firestore stories appended asynchronously
5. **Graceful degradation** — site works without Firebase; user features simply hidden
