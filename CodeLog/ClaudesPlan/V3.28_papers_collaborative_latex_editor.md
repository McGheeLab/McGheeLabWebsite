# V3.28 — Papers App: Collaborative LaTeX Editor

**Status:** Future plan (saved 2026-04-10)
**Goal:** Overleaf-like collaborative LaTeX editor integrated into the lab app ecosystem, with papers stored in GitHub repos.

## Context

The lab needs a way to simultaneously edit LaTeX documents without leaving the lab website. This app ("Papers") would sit alongside Chat, Equipment, Huddle, etc. as a full lab app. Papers live in GitHub repos for version control and peer review.

## Technology Choices

| Component | Choice | Why |
|-----------|--------|-----|
| **Editor** | CodeMirror 5 via CDN | Works with `<script>` tags (no build step). LaTeX mode available. ~400KB total. |
| **Collaboration** | Firestore presence + file-level locking | Simple, fits existing architecture. True CRDTs (Yjs) can be added later if needed. |
| **LaTeX preview** | KaTeX via CDN | Instant math rendering in-browser. Not full compilation, but fast feedback. |
| **Full compilation** | GitHub Actions (`xu-cheng/latex-action@v3`) | Full TeX Live, triggered on commit. 1-3 min latency — acceptable for final builds. |
| **GitHub integration** | GitHub REST API + Personal Access Token | No server needed. User pastes a fine-grained PAT. Stored per-user in Firestore. |
| **Real-time sync** | Firestore `onSnapshot()` | Same pattern used by all other lab apps. |

## Firestore Collections

- `papers/{paperId}` — project metadata (title, owner, collaborators, GitHub repo URL)
- `paperFiles/{fileId}` — file content + version + lock state
- `paperPresence/{paperId}_{uid}` — who's online, which file they're editing
- `paperGithubTokens/{uid}` — per-user GitHub PAT (strict auth rules: only own user can read/write)

## Phased Implementation

### Phase 1: MVP — Editor + GitHub Sync
**Files to create:**
- `apps/papers/index.html` — follow `apps/chat/index.html` pattern, add CodeMirror 5 + KaTeX CDN links
- `apps/papers/app.js` — IIFE pattern, Firebase compat SDK
- `apps/papers/styles.css` — CSS prefix `ppr-`

**Files to modify:**
- `lab-apps.js` — register Papers app in `LAB_APPS` array
- `firestore.rules` — add rules for papers collections
- `storage.rules` — add `papers/{paperId}/{allPaths=**}` path

**Features:**
1. GitHub token setup screen (paste PAT, store in `paperGithubTokens/{uid}`)
2. Create/open a paper by entering a GitHub repo URL
3. File tree sidebar showing `.tex`, `.bib`, `.sty` files from the repo
4. CodeMirror 5 editor with LaTeX syntax highlighting, bracket matching, code folding
5. KaTeX preview pane (split view, renders math environments in real-time)
6. Save = commit to GitHub via REST API (PUT `/repos/{owner}/{repo}/contents/{path}`)
7. Content cached in Firestore `paperFiles` for faster loads

### Phase 2: Multi-User Presence + Locking
1. Write presence docs to `paperPresence` when a user opens a paper
2. Display colored avatars of active collaborators in the header
3. File-level soft locks — when editing, set `lockedBy` on the file doc
4. Other users see "Alex is editing this file" banner, file becomes read-only for them
5. Auto-release locks after 60s of inactivity (heartbeat pattern)
6. Conflict detection on save — if `onSnapshot()` detects another user's changes, show merge dialog

### Phase 3: GitHub Actions Compilation
1. "Compile" button in toolbar
2. Commits all dirty files to GitHub
3. Dispatches GitHub Actions workflow via API
4. Polls for completion, downloads PDF artifact
5. Displays PDF in the app (using pdf.js or iframe)
6. Expects `.github/workflows/compile-latex.yml` in the repo (app can create it)

### Phase 4: Editor Polish
- BibTeX autocomplete for `\cite{}` commands
- File create/rename/delete from within the app
- Image upload (Firebase Storage -> commit to repo)
- Git diff viewer before committing
- Version history from git log
- LaTeX snippet toolbar (figure, table, equation, section)

## Risks
- **GitHub API rate limit (5,000/hr)**: Cache repo trees + file content in Firestore
- **Firestore 1MB doc limit**: Fine for `.tex` files (typically <100KB). Large binaries use Storage.
- **PAT security**: Per-user Firestore rules. Encourage fine-grained PATs with `repo` scope only.
- **CodeMirror 5 unmaintained**: Stable and widely deployed. Can migrate to CM6 later with a build step.
- **GitHub Actions latency (1-3 min)**: KaTeX preview provides instant math feedback for iterative work.

## Alternatives Considered
- **Just use Overleaf**: UA may have institutional access with GitHub sync. Simplest option but not integrated into lab ecosystem.
- **Self-host Overleaf**: Open source via Docker, but requires a server to maintain.
- **Typst instead of LaTeX**: Compiles to WASM (browser-viable), but journals still require LaTeX submissions.
