# V3.25 — Mobile-First Lab App Redesign

## Goal
Redesign all lab apps for phone screens with a consistent mobile shell: no hero, no traditional footer, streamlined navigation. The chat app gets a full conversation-list-first mobile experience.

## Requirements

### Shared Mobile Shell (all lab apps, ≤700px)
- **Top bar:** Filter/sort (left) | App title (center) | Hamburger menu + User icon (right)
- **Bottom bar:** Lab app quick-nav icons (left/center) | Back arrow (right)
- **No hero section, no traditional footer**
- Back arrow appears on all sub-pages (not the main list page)

### Chat App Mobile Redesign
1. **Conversation List View** (default mobile landing)
   - Flat list of all channels + DMs the user is subscribed to
   - Each item: icon, name, last message preview, timestamp, unread badge
   - Top-left filter dropdown: Newest, Most Active, Unread First, Alphabetical
   - Tapping opens the conversation detail view

2. **Conversation Detail View**
   - Full message feed (existing chat UI)
   - Top-middle stats bar: readers count (eyeball), search icon, files icon
   - Files icon opens a file list view

3. **Files View**
   - All files shared in the conversation
   - Grouped/sorted by type (images, documents, links)
   - Back arrow returns to conversation

4. **Navigation**
   - Bottom-right back arrow on all pages except conversation list
   - Bottom bar with lab app icons for quick switching

## Approach

### 1. Shared Mobile Shell (app-base.css)
- Add `.mobile-shell` layout styles
- `.mobile-top-bar` with filter, title, hamburger+user
- `.mobile-bottom-bar` with app nav icons and back arrow
- Hide `.app-header` on mobile (replaced by mobile shell)

### 2. Chat App Changes

**New state variables:**
- `_mobileView`: 'list' | 'conversation' | 'files' | 'stats'
- `_conversationFilter`: 'newest' | 'active' | 'unread' | 'alpha'

**New render functions:**
- `renderMobileConversationList()` — flat channel/DM list
- `renderMobileStatsBar()` — readers, search, files
- `renderMobileFilesView()` — files grouped by type
- `renderMobileBottomBar()` — app nav + back arrow

**CSS additions:**
- `.chat-conv-list` — conversation list styles
- `.chat-conv-item` — individual conversation card
- `.chat-mobile-stats` — stats bar
- `.chat-files-view` — files grouped by type
- `.mobile-bottom-bar` — bottom navigation

### 3. Other Lab Apps
- Wrap existing mobile breakpoints with the shared shell
- Add bottom bar navigation to each app

## Files to Modify
- `apps/shared/app-base.css` — shared mobile shell
- `apps/chat/styles.css` — conversation list, files view, stats bar
- `apps/chat/app.js` — new mobile views and navigation
- `apps/chat/index.html` — (minor, if needed)
- Other app CSS files — add bottom bar

## Key Decisions
- Mobile detection via CSS `@media (max-width: 700px)` and JS `window.innerWidth <= 700`
- Conversation list is mobile-only; desktop keeps existing sidebar layout
- Back arrow uses browser-style navigation stack (not hash routing)
