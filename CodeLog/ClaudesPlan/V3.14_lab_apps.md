# V3.14 — Lab Apps Hub (Private Section)

## Goal
Add a new private, authenticated section of the website that serves as a hub for internal lab applications. This section is **not visible to guests or unauthenticated visitors**. Only logged-in lab members can see and access it; admins get additional privileges.

## Apps
1. **Inventory Tracker** (`apps/inventory/`) — Lab supply and equipment inventory management
2. **Equipment Scheduler** (`apps/equipment/`) — Google Calendar integration for booking microscopes and shared equipment
3. **Lab Meeting** (`apps/meetings/`) — Agenda management, presentation scheduling, notes for lab meetings
4. **Admin Console** (`apps/console/`) — Master control panel for all lab apps (admin-only)

## Architecture: Standalone Apps

Each app is a **self-contained mini-application** that can run independently:

```
apps/
├── shared/
│   ├── auth-bridge.js    # postMessage auth handshake (embedded ↔ standalone)
│   └── app-base.css      # Dark theme base styles, common components
├── inventory/
│   ├── index.html         # Standalone entry point (loads Firebase, bridge, own JS)
│   ├── app.js             # App logic — inits via AppBridge.onReady()
│   └── styles.css         # App-specific styles
├── equipment/
│   ├── index.html
│   ├── app.js
│   └── styles.css
├── meetings/
│   ├── index.html
│   ├── app.js
│   └── styles.css
└── console/
    ├── index.html
    ├── app.js
    └── styles.css
```

### Two execution modes

1. **Embedded** (accessed via `#/apps/{id}` on main site)
   - `lab-apps.js` renders an `<iframe>` pointing to `apps/{id}/index.html`
   - Parent sends auth via `postMessage` → child's `auth-bridge.js` receives it
   - App header is hidden (`.app-embedded .app-header { display: none }`)
   - iframe auto-resizes via `mcgheelab-app-resize` messages

2. **Standalone** (opened directly at `apps/{id}/index.html`)
   - Loads Firebase SDK + `firebase-config.js` directly
   - `auth-bridge.js` detects `window.parent === window` → uses `onAuthStateChanged`
   - App header visible with "Back to Lab Apps" link
   - Full standalone page with its own `<head>`, styles, etc.

### Auth bridge protocol
- Child sends `{ type: 'mcgheelab-app-ready' }` on load
- Parent responds with `{ type: 'mcgheelab-auth', user, profile }`
- Child fires `AppBridge.onReady(user, profile)` → app renders
- Timeout: 5s without auth → shows sign-in wall

## Approach
- `lab-apps.js` — Hub renderer (card grid) + iframe embedder with postMessage auth bridge
- `apps/shared/auth-bridge.js` — Dual-mode auth (embedded: postMessage, standalone: Firebase direct)
- `apps/shared/app-base.css` — Common dark-theme styles, buttons, inputs, cards, auth wall
- Each app: `index.html` + `app.js` + `styles.css` — fully self-contained
- Hash routes: `#/apps` (hub), `#/apps/{id}` (embedded iframe)
- Nav: "Lab Apps" visible to authenticated non-guest users

## Key Decisions
- **Standalone-first**: Each app is a real HTML page, not just a JS module. Can be developed/tested by opening its `index.html` directly.
- **Iframe isolation**: Apps are sandboxed in the main site. No namespace collisions, independent CSS, own error boundaries.
- **postMessage auth**: No custom tokens needed (would require Cloud Functions). Parent passes user/profile object directly. Same-origin policy protects the handshake.
- **Shared base**: `app-base.css` ensures visual consistency without coupling apps to the main site's stylesheet.
