# V3.5 — Firestore-Driven Classes Listing

## Goal
Remove hardcoded class entries from `content.json` and load the classes listing page from a Firestore `classes` collection. Admin creates/manages course listings through the dashboard.

## Approach

### New Firestore Collection: `classes`
```
{
  title: "BME 295C",
  description: "Research seminar series...",
  level: "Graduate",
  when: "Summer 2026",
  registrationLink: "",
  detailPage: "bme295c",       // optional — links to schedule page
  status: "published",         // published | draft
  order: 0,                    // display order
  createdAt: serverTimestamp(),
  updatedAt: serverTimestamp()
}
```

### Changes

1. **content.json** — Remove `classes.courses` array, keep `classes.intro`
2. **firestore.rules** — Add `classes` collection (public read, admin write)
3. **user-system.js** — Add DB methods: `getPublishedClasses()`, `getAllClasses()`, `saveClass()`, `deleteClass()`
4. **user-system.js** — Add "Course Listings" form + list in admin Classes tab
5. **app.js** — Update `renderClasses()` to show loading placeholder; add Firestore loading in wiring section

### Key Decisions
- Separate `classes` collection from `schedules` — not all courses need a class-builder page
- `detailPage` field optionally links a listing to a schedule (class-builder page)
- Admin manages listings via dashboard; no content.json editing needed
