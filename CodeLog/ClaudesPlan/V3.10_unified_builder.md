# V3.10 — Unified Builder: Side-by-Side Calendar + Day-Column Time Grid

## Goal
Both session and freeform modes share the same layout: calendar on left, scrollable time grid on right. The time grid has one column per selected day with date headers and clear column demarcations. The difference between modes is how the user interacts with the grid.

## Two Modes, Same Grid

### Freeform
- Click/drag individual cells to toggle availability (paint mode like when2meet)
- Each cell is independently selectable
- Stored as: `selectedDays` + time range + granularity (existing model)

### Sessions
- Admin sets session duration first (e.g., 30 min, 1 hr, 1.5 hr, 2 hr)
- Click a cell to place a block of that duration starting at that time
- Block snaps to 15-min grid, spans `duration / 15` rows
- Click an existing block to remove it
- Each block is per-day (not applied to all days)
- Stored as: `sessionBlocks: [{ id, day, start, end, color }]`
  - Note: `day` is a single ISO date string (not `days` array)

## Data Model Changes

### Session blocks (updated)
```javascript
// OLD: one block applies to multiple days
{ id, days: ['2026-06-08', '2026-06-10'], start, end, color }

// NEW: one block per day (more granular, matches the per-cell UI)
{ id, day: '2026-06-08', start: '09:00', end: '10:00', color }
```

Color assignment: sequential index, wraps around palette.

### Freeform (unchanged)
```javascript
{ selectedDays, startHour, endHour, granularity }
```

## Session Duration Selector
Dropdown above the time grid in session mode:
- 15 min, 30 min, 45 min, 1 hour, 1.5 hours, 2 hours
- Default: 1 hour
- Stored on schedule as `sessionDuration` (minutes)
