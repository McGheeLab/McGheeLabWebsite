# V3.26 — Multi-Day Equipment Booking + Calendar Freeze-Pane & Zoom

## Goal
1. Allow equipment bookings to span multiple days, with both start and end having a date + time.
2. Make the calendar toolbar, legend, and day headers stay fixed ("freeze-pane") while only the time grid scrolls.
3. Add zoom controls so users can see the full 24h day or zoom into specific hours.

## Approach

### Multi-Day Bookings
- Add `endDate` field to booking data model (keeps `date` as start date for backward compatibility)
- Restructure booking modal: "Start Date / Start Time" row + "End Date / End Time" row
- Validation: end date must be >= start date; same-day bookings still require end time > start time
- Duration calculated across day boundaries: `(dayDiff * 24 * 60) + endTimeMin - startTimeMin`

### Calendar Freeze-Pane Layout
- `.eq-layout` wraps tabs + content in a flex column filling the viewport
- `eq-cal-header` (toolbar + legend-row) is flex-shrink: 0 — never scrolls
- `eq-cal-body` fills remaining space with overflow: hidden
- `.eq-cal-scroll-area` flex row holds `eq-week-wrap` + `eq-time-slider` side by side
- `.eq-week-wrap` is the sole scroll container; day headers use `position: sticky; top: 0`; native scrollbar hidden
- Auto-scroll to 8 AM on first render; scroll position preserved proportionally on re-render

### Custom Time Slider
- Built into the week view as `eq-time-slider` element alongside `eq-week-wrap`
- Spans only the time grid height (not toolbar, legend, or day headers)
- Thumb height proportional to `clientHeight / scrollHeight` — scales with zoom
- Auto-hides when zoomed out far enough that all content fits
- Replaces mobile-shell `enableTimeScroll` (calls `disableTimeScroll` on render)

### Time-Scale Zoom
- 9 zoom levels: [4, 6, 9, 14, 20, 28, 40, 56, 76] px per half-hour slot (default index 4 = 20px)
- At minimum zoom (4px/slot), full 24h day is 192px — fits any screen
- +/− buttons in the legend row, right-aligned (above Sunday column)
- Pinch-to-zoom on touch devices with ratio-based thresholds
- `grid-template-rows` uses explicit `${slotH}px` instead of `1fr`

## Files Modified
- `apps/equipment/app.js` — booking modal, validation, conflict checking, week/month view rendering, detail popup, My Bookings, subscription query, Google Calendar sync, zoom state, pinch gesture, scroll management, custom time slider, wireTimeSlider()
- `apps/equipment/styles.css` — flex layout, sticky headers, scroll-area flex row, custom slider styles, legend-row, zoom controls, hidden native scrollbar, multi-day block segments, multi-day badge

## Key Decisions
1. **Backward compatible**: existing single-day bookings work unchanged (`endDate` defaults to `date` when absent)
2. **Week view segments**: multi-day bookings render as separate blocks per visible day (first day: startTime→23:30, middle days: 00:00→23:30, last day: 00:00→endTime) — same approach as Google Calendar
3. **Subscription widening**: query window extended 30 days before view start to catch multi-day bookings that began earlier but still span into the view
4. **Conflict checking**: queries bookings where `date <= newEndDate`, then filters client-side for date range + time overlap
5. **Google Calendar sync**: uses `date` for event start and `endDate` for event end, so GCal shows the full span
6. **Sticky headers via CSS**: `position: sticky; top: 0` on grid row 1 cells, inside a scroll wrapper — no JS needed for the freeze effect
7. **`:has()` selector**: `#eq-content:has(.eq-cal-body)` disables parent scrolling only when calendar is active, so other tabs scroll normally
8. **Custom slider over mobile-shell**: built inline rather than using the fixed-position mobile-shell slider, so it precisely aligns with the time grid and its thumb scales with zoom
