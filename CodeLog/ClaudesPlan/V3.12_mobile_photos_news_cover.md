# V3.12 — Mobile Team Photos & News Cover Images

## Goal
1. Make team member headshots larger and more visible on mobile screens
2. Add cover image support to news posts

## Approach

### Mobile Team Photos
- On screens ≤ 480px, switch `.person-layout` from side-by-side (80px photo + info) to a stacked vertical layout with a centered 120px photo
- PI cards already stacked on ≤ 600px but now center the photo and info for better visual balance
- No JS changes needed — pure CSS media query adjustment

### News Cover Images
- **Display** (`app.js`): `buildNewsFeedCard()` reads `p.coverImage` (same `{ thumb, medium, full }` structure used by projects) and renders a horizontal grid layout (`.news-cover-layout`) — image left (280px), text right. Stacks vertically on mobile ≤ 600px. Cards without cover images render unchanged.
- **Editor** (`user-system.js`): Add a cover image upload zone before the sections block in `renderNewsEditor()`. Wire drag-and-drop and click-to-upload using existing `processImage()` + `uploadImageSet()` pipeline. Store as `coverImageUrls`, load existing cover when editing.
- **Data**: `collectData()` includes `coverImage` in the Firestore document
- **CSS** (`user-styles.css`): `.story-feed-cover` — full-width, max-height 320px, object-fit cover, rounded corners

## Files Modified
- `styles.css` — Mobile media queries for `.person-layout` and `.pi-layout`
- `app.js` — `buildNewsFeedCard()` cover image rendering
- `user-system.js` — News editor cover image upload (HTML + wiring + data)
- `user-styles.css` — `.story-feed-cover` styles

## Key Decisions
- Reuse the same `processImage()` / `uploadImageSet()` pipeline that projects and story sections use, ensuring consistent multi-resolution image handling
- Cover image appears below the title and above the description, matching common news/blog card patterns
- 120px photo on mobile chosen as a good balance — large enough to clearly see facial features while leaving room for name/role text
