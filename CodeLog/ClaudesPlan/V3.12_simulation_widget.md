# V3.12 — Simulation Widget

## Goal
Add a new **Simulation** widget to the classes editor that runs interactive HTML/JS simulations in a sandboxed iframe, with student code editing, fullscreen mode, and data export.

## Requirements
1. **Admin places simulation code** — HTML+JS textarea saved to Firestore
2. **Students can edit code temporarily** — Local copy only, never persists back to source
3. **Fullscreen mode** — Fullscreen API on the iframe container
4. **Data export** — Simulation can call `SimExport.log(label, value)` to register data; parent provides CSV download

## Approach

### Widget Data Structure
```javascript
{ kind: 'simulation', id: 'blk_xxx', code: '<html>...</html>', title: '' }
```

### Sandboxed Execution
- Use `<iframe srcdoc="...">` with `sandbox="allow-scripts"` — scripts run but cannot access parent DOM, cookies, or navigation
- Inject a `SimExport` bridge into the iframe via a `<script>` preamble that uses `postMessage` to send data to parent
- Parent window listens for `message` events from simulation iframes

### Student Code Editing
- View mode shows a "Edit Code" toggle that reveals a `<textarea>` pre-filled with the source
- Edits modify only the local iframe `srcdoc` — a "Run" button re-renders the iframe
- "Reset" button restores the original admin-authored code
- Changes are never written back to Firestore

### Fullscreen
- Container div wraps the iframe + toolbar
- Fullscreen button calls `container.requestFullscreen()`

### Data Export
- Preamble script injected into iframe:
  ```javascript
  window.SimExport = {
    _data: [],
    log(label, value) { this._data.push({label, value, t: Date.now()}); },
    _flush() { parent.postMessage({type:'sim-export', data: this._data}, '*'); }
  };
  ```
- "Export Data" button sends a message to the iframe to flush, then parent receives and converts to CSV download
