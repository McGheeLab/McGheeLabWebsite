# V3.45 — Retire `/apps/purchases/` + `/apps/procurement/`

## Goal

Delete the two student-submission lab apps that V3.44's unified
[Procurement page](../../rm/pages/procurement.html) replaces. Both lab
apps are now redundant — the new page submits to a different collection
(`procurementTickets`) and provides the full lifecycle (request →
order → receive → place → inventory) in one place. With V3.44 deployed
to production (Firestore + Storage rules merged, static tree FTP'd),
the burn-in window is over and these can go.

## What's deleted

- **`/apps/purchases/`** — 126 LOC, wrote to `purchaseRequests`
  collection. Submission form for "I need this thing, PI please order
  it." Read by [rm/pages/purchase-requests.html](../../rm/pages/purchase-requests.html)
  for admin review.
- **`/apps/procurement/`** — 337 LOC, wrote to `procurement`
  collection. Submission form for "I bought this thing on my pcard,
  here's the receipt + line items + funding source." Read by
  [rm/pages/receipts.html](../../rm/pages/receipts.html) for the
  Incoming tab.

## What stays

- **`purchaseRequests` collection** — preserved (read-only going
  forward). Existing data is still visible in
  [rm/pages/purchase-requests.html](../../rm/pages/purchase-requests.html).
  No new writes after V3.45; if a future phase wants to migrate the
  historical data into `procurementTickets`, that's a separate sweep.
- **`procurement` collection** — preserved (read-only). Existing data
  visible in [rm/pages/receipts.html](../../rm/pages/receipts.html)'s
  Incoming tab.
- **`rm/pages/purchase-requests.html`** + **`rm/pages/receipts.html`**
  — both unchanged. Nav still surfaces them as `Purchase Requests` and
  `Receipts & Budget` in the Lab Admin group. They've always been
  admin-side review pages and remain useful for the legacy collections.
- **`firestore.rules`** for `purchaseRequests` and `procurement` —
  unchanged. Read access still works for the admin pages.

## Approach

1. `git rm -rf apps/purchases/ apps/procurement/` — drops 6 files
   total (app.js + index.html + styles.css × 2). The pre-existing
   local cache-buster modifications (`?v=2` query params on the
   index.html files) are discarded with the directories.
2. Remove the two entries (`id: 'procurement'` and `id: 'purchases'`)
   from the `LAB_APPS` registry in [lab-apps.js](../../lab-apps.js)
   (was lines 117–141). The registry is already orphaned in the
   public nav (V3.40 retired the Apps menu); keeping it consistent
   matters because Phase C will delete the file outright.
3. `sw.js` `CACHE_VERSION` 19 → 20.
4. CHANGELOG + ARCHITECTURE updates.
