# V3.43 — Inventory: delete the stub

## Goal

Delete `/apps/inventory/`. Per the V3.40 audit, the lab app is a 48-line
"Coming Soon" placeholder identical in shape to the V3.42-deleted console
stub: render three section cards (Supplies / Equipment / Orders) plus an
`Under Development` badge, no Firestore reads/writes, no event handlers
(`wire()` is empty). Meanwhile [rm/pages/inventory.html](../../rm/pages/inventory.html)
is the real, 903-LOC inventory management system: 8 top-level tabs, 30+
subcategories, multi-location stock, search/filter/sort, in-line editing,
LIVE_SYNC, "Create Group" feature.

## Reorder note

V3.43 was originally going to be `purchases`. Audit revealed a deeper
problem: the RM admin pages for purchases, procurement, and compliance
have **no student-submission form**. The lab apps are the only path
students have today to file purchase requests / submit receipts /
submit training certificates. Deleting them outright would create three
parallel feature regressions. Each requires an `+ Submit` form to land
on the corresponding RM admin page first — a meaningfully bigger lift
than V3.42 / V3.43.

Inventory is the rare overlap-app where the lab version is a **pure
stub**: nothing to lose by deleting it. Slotted V3.43 = inventory and
deferred V3.44–V3.46 (purchases / procurement / compliance) until the
submission flows are added. Final remainder order:

| V | App | Status |
|---|---|---|
| V3.42 | console | deleted (V3.42) |
| **V3.43** | **inventory** | **delete (this release)** |
| V3.44 | purchases | + Submit Request form on RM page, then delete |
| V3.45 | procurement | + Submit form on RM page, then delete |
| V3.46 | compliance | + Submit Certificate form on RM page, then delete |
| V3.47 | scheduler | subset port |
| V3.48 | settings | overlap, splits user/lab scope |
| V3.49 | activity-tracker | replaces existing iframe wrapper |
| V3.50 | huddle | multi-path LIVE_SYNC |
| V3.51 | equipment | OAuth refactor + scheduler My Schedule re-port |
| V3.52 | chat | largest, real-time fanout |

## What's in the stub

[apps/inventory/app.js](../../apps/inventory/app.js) (48 LOC) renders
three "section" cards with description text and an `Under Development`
badge. `wire()` is empty. No Firestore touch, no AppBridge data calls.
The Phase A nav (Lab Admin group) already points at the real
[rm/pages/inventory.html](../../rm/pages/inventory.html), so deletion is
a strict subtraction.

## Approach

1. `git rm -rf apps/inventory/` — drops `app.js` (48 LOC), `index.html`,
   `styles.css`. The pre-existing local cache-buster modifications to
   `apps/inventory/index.html` (`?v=2`) are discarded with the directory.
2. Remove the `inventory` entry from the `LAB_APPS` registry in
   [lab-apps.js](../../lab-apps.js) (lines 12–24).
3. `sw.js` `CACHE_VERSION` 17 → 18.
4. CHANGELOG + ARCHITECTURE updates.
