# V3.55 — Scheduler: My Schedule tab (closes V3.47 deferred work)

## Goal

Lift the "My Schedule" half of `/apps/scheduler/app.js` into
[rm/pages/scheduler.html](../../rm/pages/scheduler.html) as a new
tab alongside the existing My Schedulers (V3.47). With
ScheduleService + CalendarService + ScheduleUtils brought into RM
in V3.51, the personal calendar with layered availability + Google
Calendar overlay can finally light up natively.

## What lands

A new `My Schedule` tab at the top of [rm/pages/scheduler.html](../../rm/pages/scheduler.html):

- **Weekly time-grid Gantt** — 24-hour scrollable grid, 7 days
  across, 30-min slots, zoom in/out (9 zoom levels via
  ScheduleUtils.ZOOM_LEVELS).
- **Mode selector** — General Availability (green) / Special
  Availability (blue) / Special Unavailability (red). Drag on the
  grid to create blocks in the active mode.
- **Layer panel** — toggles for which sources to overlay:
  General Availability (recurring template), Special Availability /
  Unavailability (per-week overrides), External Calendars (Google /
  Outlook / Apple, dynamically named from CalendarService.isConnected),
  Custom Events. "Restore N dismissed event(s)" button when any
  calendar events have been dismissed.
- **Calendar event integration** — events from connected calendars
  overlay the Gantt as purple/green blocks. Per-event status toggle
  (unavailable ↔ busy-but-available) and dismiss button. Connection
  setup happens from the V3.54 Settings → Calendar Sync tab.
- **Custom events** — `+ Custom Event` button opens a modal to
  create one-off events with title, color, time range. Shown in the
  Custom Events layer with a per-block edit modal on click.
- **Block editor modals**:
  - `showNewScheduleBlockModal` — opens after drag-create with
    fields appropriate to the active mode (mode + time range + reason
    + rigidity for blackout, mode+time for availability).
  - `showScheduleBlockEditor` — opens on click of an existing
    template/override block; lets user change reason / rigidity /
    delete.
  - `showCustomEventModal` — create or edit custom events.
  - `showEditCustomEventModal` — small wrapper that pulls the
    current event and opens the modal pre-filled.
- **Week navigation** — prev / next week arrows + a "Today" button
  that appears when `_weekOffset !== 0`.
- **Toolbar** — `Copy Mon → Weekdays` (replicates Monday's blocks
  to Tue–Fri in the recurring template) + zoom controls.

## Default tab

`_currentTab = 'schedulers'` — preserves the V3.47 behavior (page
loads on My Schedulers). User clicks `My Schedule` in the top tab
bar to switch. This avoids surprising V3.47 users while still
making the new tab fully functional.

## Implementation

### `rm/js/scheduler.js`

Grew from 462 LOC to **1,246 LOC**. Three changes:

1. **State + service aliases** at the top:
   - `_currentTab` (default `'schedulers'`).
   - `_weekOffset`, `_schedMode`, `_schedZoomIdx` for My Schedule.
   - `SS()` / `SU()` / `CS()` lazy lookups for `McgheeLab.ScheduleService`
     / `.ScheduleUtils` / `.CalendarService` (null when not loaded).
   - `lockIcon()` / `calIcon()` helpers.
2. **Render dispatcher** rewritten:
   - `render()` now emits a tab bar + a `<div id="sched-tab-content">`
     container, then dispatches to `renderSchedulersTab(container)`
     (existing V3.47 code, pulled out of the old `render()` body) or
     `renderMyScheduleTab(container)` (new, lifted from lab app).
   - Tab switch resets `_view='list'` when entering Schedulers so the
     editor doesn't resume into a half-loaded state.
3. **Boot extension** — after `firebridge.whenAuthResolved()`, init
   ScheduleService + CalendarService (each guarded `if (SS())` /
   `if (CS())` so they no-op when scripts aren't loaded). Subscribe
   to their `onChange` callbacks: re-render the My Schedule content
   only when `_currentTab === 'myschedule'`.
4. **My Schedule code injected before the IIFE close** — 720 LOC
   lifted verbatim from `/apps/scheduler/app.js` lines 191–908 (the
   complete `renderMyScheduleTab` / `renderMyScheduleContent` /
   `renderMyScheduleHTML` / `wireMySchedule` + four block-editor /
   custom-event modal helpers). Patches applied via `sed`:
   - `appEl.querySelector(All)` → `root.querySelector(All)`. (Other
     references to `appEl` were absent because the renderer scopes
     queries to `root`-rooted DOM nodes.)
   - Trailing section divider stripped.

### `rm/css/scheduler.css`

**Unchanged.** The V3.47 lift was a verbatim copy of `/apps/scheduler/styles.css`,
which already included the `.ms-*` selectors used by the My Schedule
tab. The plan doc explicitly noted at the time: *"Includes `.ms-*`
classes that the deferred My Schedule tab uses — they ride along
now so V3.51 doesn't have to re-import the CSS."* (Held true a
release later — V3.55 in this case.)

### `rm/pages/scheduler.html`

**Unchanged in V3.55.** V3.51 already added the `<script>` tags
for `schedule-utils.js`, `schedule-service.js`, `calendar-service.js`,
plus the GIS client. With the new renderer in `rm/js/scheduler.js`
expecting those services on `McgheeLab.X`, everything wires up
without HTML changes.

### `sw.js`

`CACHE_VERSION` 29 → 30.
