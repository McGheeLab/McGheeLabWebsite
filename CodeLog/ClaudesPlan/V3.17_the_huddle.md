# The Huddle — Lab Weekly Planning App

## Context

Lab members need a low-friction way to share what they're working on each week so others can sign up to watch or join protocols/methods. This replaces ad-hoc "what are you doing?" conversations with a persistent, community-driven planning board. The key insight from researching tools like Geekbot, Range, and protocols.io is that **the tool must be easier than not using it** — so smart suggestions, one-tap actions, and ambient awareness are critical.

---

## Feature Set (Organized by Phase)

### Phase 1: Core MVP — Useful on Day 1

| # | Feature | What It Does | Why Students Use It |
|---|---------|-------------|-------------------|
| 1.1 | **Weekly Plan Board** | Week-at-a-glance feed showing every member's planned tasks. Cards grouped by day or by member (toggle). Week navigation (prev/next/today). | One place to see what the whole lab is doing. Replaces "what are you up to?" |
| 1.2 | **Watch/Join Sign-ups** | Click "Watch" (observe) or "Join" (participate) on any task. Owner sees who signed up. | Removes the awkwardness of asking to observe. Creates natural peer-learning. |
| 1.3 | **Protocol Linking** | Attach a protocol (URL + title) to any task. Clickable chip on the card so watchers can pre-read. | Watchers/joiners arrive prepared. Builds a practical link between plans and reference materials. |
| 1.4 | **Weekly Check-in Prompt** | If no plans posted this week, a guided prompt appears: "What protocols are you running?", "Anything blocking you?", "Need help with anything?" | Lowers friction vs. blank form. Blockers/help fields create mentoring opportunities. |
| 1.5 | **My Plans View** | Personal view of your own tasks. Add/edit/delete plans. Mark as completed/skipped. | Quick self-management without noise from the full feed. |

### Phase 2: Sticky & Smart

| # | Feature | What It Does | Why Students Use It |
|---|---------|-------------|-------------------|
| 2.1 | **Smart Suggestions** | Reads your Activity Tracker history to suggest recurring tasks as one-tap chips ("You logged 'cell culture' 8 times last month — add it?"). | Most lab work is recurring. One tap instead of typing. |
| 2.2 | **Skill Level Tiers** | When watching/joining, select: Learning (first time), Practiced (done a few times), or Can Teach (expert). Color-coded badges. | Task owner knows if their watcher needs guidance or can help. Lab-wide skill visibility. |
| 2.3 | **Protocol Library** | Shared, searchable lab-wide protocol list. Grows organically — every linked protocol optionally joins the library. Usage count, tags, author. | Central place to find protocols. Discovery is organic. |
| 2.4 | **Project Tags** | Tag plans with a project (from existing `projectPackages`). Filter the feed by project. | PI can filter by research project. Helps the system learn what you're working on. |
| 2.5 | **Weekly Digest** | Auto-compiled read-only summary for each closed week: planned vs. completed, who watched/joined, lab-wide stats. Generated lazily (no Cloud Functions). | Accountability without micromanagement. PI gets a snapshot. |

### Phase 2.5: Status Management

| # | Feature | What It Does | Why Students Use It |
|---|---------|-------------|-------------------|
| 2.6 | **Cancelled / Delayed Status** | Plans can be marked as "cancelled" (won't happen) or "delayed" (pushed to a future week). Cancelled shows strikethrough + reason field. Delayed prompts to pick a new target week and auto-creates a copy in that week. Watchers/joiners get a visual indicator so they know not to show up. | Lab work rarely goes as planned — reagent delays, equipment issues, etc. This keeps the board honest and communicates changes to anyone who signed up. |

### Phase 3: Long-term Engagement

| # | Feature | What It Does | Why Students Use It |
|---|---------|-------------|-------------------|
| 3.1 | **AI Plan Generator** | Uses Anthropic API (same pattern as Activity Tracker) to draft a weekly plan from your history, project, and patterns. Editable before posting. | "Write my weekly plan for me" — ultimate low-friction feature. |
| 3.2 | **Kudos / Peer Recognition** | After a watch/join session, leave quick kudos: "Great explanation", "Learned a lot". Accumulates on profiles. | Social reinforcement for teaching/mentoring. Positive feedback loop. |
| 3.3 | **Streaks & Badges** | Track posting consistency (streak counter). Milestone badges: "First Plan", "Mentored 5 People", "10-Week Streak". Visible on profile cards. | Gamification drives habit formation. The 2-3 week adoption window is critical. |
| 3.4 | **Skill Tree / Proficiency Map** | Aggregates watch/join data into a per-student protocol skill map. Grid showing learned/practiced/can-teach per protocol. | Students see growth. PI sees capability gaps. Useful for onboarding. |

## Data Model (Firestore Collections)

### `huddlePlans/{planId}` — Core collection (one doc per task)

```javascript
{
  ownerUid:      string,        // Firebase Auth UID
  ownerName:     string,        // Denormalized for feed rendering
  ownerCategory: string,        // 'grad', 'undergrad', etc.
  weekId:        string,        // ISO week: '2026-W15'
  plannedDay:    string|null,   // ISO date: '2026-04-13'
  text:          string,        // "Cell culture passage for PDMS experiment"
  notes:         string,        // Optional longer description
  protocol:      { title, url, protocolId } | null,
  projectId:     string|null,
  projectName:   string|null,   // Denormalized
  watchers:      [{ uid, name, skillLevel }],  // Embedded array
  joiners:       [{ uid, name, skillLevel }],
  status:        'planned' | 'completed' | 'skipped' | 'cancelled' | 'delayed',
  statusReason:  string|null,    // Why cancelled/delayed (optional note)
  delayedToWeek: string|null,    // If delayed, the target weekId
  createdAt:     Timestamp,
  updatedAt:     Timestamp
}
```

### `huddleCheckins/{checkinId}` — Weekly check-in responses

```javascript
{
  uid, weekId, plansText, blockers, helpNeeded, createdAt, updatedAt
}
```

### `huddleProtocols/{protocolId}` — Shared protocol library (Phase 2)

```javascript
{
  title, url, tags[], authorUid, authorName, usageCount, createdAt, updatedAt
}
```

### `huddleDigests/{weekId}` — Weekly summaries (Phase 2)

### `huddleStats/{uid}` — Streaks, badges, skill map (Phase 3)

### `huddleKudos/{kudosId}` — Peer recognition (Phase 3)

**Key design decisions:**
- Watchers/joiners are embedded arrays (not subcollections) because lab is <20 people. Uses `arrayUnion`/`arrayRemove` for concurrent sign-ups.
- Denormalized names on plan docs to avoid N+1 reads on the feed.
- Single query `where('weekId', '==', currentWeek)` fetches the entire week for the whole lab.
- Uses `onSnapshot` for real-time feed updates (unlike Activity Tracker which uses one-time reads) since this is a collaborative tool.

## Verification Plan

1. **Local test**: `python3 -m http.server 8000` from repo root, navigate to `#/apps/huddle`
2. **Auth**: Verify AppBridge fires and user/profile are available
3. **CRUD**: Create a plan, verify it appears in the feed, edit it, delete it
4. **Watch/Join**: Log in as a second user, watch a plan, verify the owner sees the watcher
5. **Real-time**: Open two browser tabs, create a plan in one, verify it appears in the other
6. **Mobile**: Test at 375px width — all interactions must be touch-friendly
7. **Embedded mode**: Navigate via `#/apps/huddle` on main site, verify iframe auth and layout
