# V3.2 — CV Builder for Students

## Goal
Port the full-featured CV Manager from `McGheeLab/CV` (React SPA) to the McGheeLab website as a vanilla JS page. Users can build, manage, import, export, and version their academic CVs. CV entries sync bidirectionally with user profile associations (papers, presentations, patents, etc.).

## Source Reference
- **Repo:** https://github.com/McGheeLab/CV
- **App:** `professor-cv/src/App.jsx` (~2,444 lines, React 19 + Vite + recharts)
- **Storage:** Currently localStorage (`cv-data-v4`) — we migrate to Firestore

## Architecture Decisions

### Storage: Firestore `cvData/{userId}`
- One document per user containing the full CV data object
- Schema matches the React app's `mkDefault()`: profile, journals, conferences, books, patents, presentations, grants, students, awards, service, courses, software, cv_versions, citation_meta
- CV profile fields auto-populate from user's Firestore profile (name, email, bio, etc.)

### New File: `cv-builder.js`
- Separate file (~3,000+ lines) to avoid bloating `user-system.js` (already 3,929 lines)
- Follows existing pattern: uses `window.McgheeLab` namespace
- Exports `renderCV()` and `wireCV()` functions consumed by `app.js`

### New File: `cv-styles.css`
- All CV builder styling in dedicated stylesheet
- Dark theme matching the existing site (gold accent `#C9A84C`)
- Responsive/mobile-first

### Charts: Chart.js via CDN
- Replaces recharts (React-only)
- Lightweight, no framework dependency
- Used for analytics: pubs/year, citations/year, quartile distribution

### Association Sync
Bidirectional mapping between CV sections and user profile fields:

| CV Section | User Profile Field | Sync |
|---|---|---|
| journals | papers | title + DOI → {title, url} |
| conferences | posters | title + DOI → {title, url} |
| presentations | presentations | title + event → {title, url} |
| patents | patents | title + number → {title, url} |
| — | protocols | No CV equivalent (kept separate) |
| — | finalWork | No CV equivalent (kept separate) |

On CV save: update user profile associations automatically.
On profile load: if CV is empty but profile has associations, offer to import them.

### Access Control
- All authenticated non-guest users can access `#/cv`
- Each user can only read/write their own CV document
- Admin can read any CV (for oversight)

### AI Assistant
- Per-user API key stored in their Firestore user profile (`anthropicKey` field)
- Key is encrypted/masked in UI, stored as-is in Firestore (private to user)
- Uses Claude API directly from browser (with `anthropic-dangerous-direct-browser-access` header)

## Features (Full Port)

### Overview Group
1. **Dashboard** — Publication/citation/funding summary stats, section counts, quick navigation
2. **Analytics** — Charts: pubs/year, citations/year, quartile distribution, citation distribution (Chart.js)
3. **Profile** — Edit CV profile info (auto-populated from Firestore user profile)

### Publications Group
4. **Journal Papers** — CRUD, DOI auto-fill (CrossRef), search/filter, status badges
5. **Conference Papers** — Same pattern as journals
6. **Books & Chapters** — Same pattern

### Research Group
7. **Patents** — CRUD with patent-specific fields
8. **Grants & Funding** — CRUD with funding amounts, date ranges, roles
9. **Software & Data** — CRUD for tools, datasets, algorithms

### Activities Group
10. **Presentations** — CRUD for talks, seminars, keynotes
11. **Awards & Honors** — CRUD with categories
12. **Service & Editorial** — CRUD with ongoing flag

### Teaching Group
13. **Students** — CRUD for supervised students
14. **Courses Taught** — CRUD with nested TCE evaluations

### Tools Group
15. **Import** — PDF upload (NSF C&P, Biosketch parsing), BibTeX file upload, paste BibTeX, TCE report PDFs
16. **AI Assistant** — Summarize abstract, impact statement, research bio, cover letter, keywords
17. **CV Versions** — Named versions with section/entry visibility toggles
18. **Export & Share** — PDF (Classic/Modern/NSF templates), LaTeX (moderncv/plain), BibTeX, plain text
19. **Citation Tracker** — CrossRef citation count updates with scheduling
20. **ORCID Sync** — Fetch works from ORCID public API
21. **Settings** — API key management, data backup/restore (JSON export/import from Firestore)
