# V3.19 — Scheduler Lab App

## Goal
Move the scheduler from the dashboard into a standalone lab app, preserving all existing functionality while making it accessible from the Lab Apps hub.

## Approach
Extract the scheduler management UI (list, create, edit) from `user-system.js` dashboard into a self-contained app at `apps/scheduler/`. The app reuses the existing `scheduler.js` rendering engine via `<script>` tag and includes its own copy of `ScheduleDB` for Firestore operations.

## Files Modified
- `lab-apps.js` — Added scheduler to `LAB_APPS` registry with `status: 'active'`
- `CodeLog/Updates/CHANGELOG.md` — V3.19 entry
- `CodeLog/Architecture/ARCHITECTURE.md` — Updated app list and added scheduler app description

## Files Created
- `apps/scheduler/index.html` — HTML shell with Firebase SDK, auth bridge, scheduler.js engine, and app.js
- `apps/scheduler/styles.css` — All scheduler-related CSS (calendar, time grid, builder, session list, speaker forms, freeform grid, mode toggle, admin table, confirmed speakers)
- `apps/scheduler/app.js` — Standalone app logic: ScheduleDB, list view, create form, editor view using McgheeLab.Scheduler engine

## Key Decisions
- **Load scheduler.js via script tag** rather than inlining — keeps the engine as a single source of truth shared with class-builder and other consumers
- **Self-contained ScheduleDB** in app.js — avoids loading the entire class-builder.js (which has course builder, section registry, widget system, etc.)
- **Invite URLs point to main site** (`#/schedule/{id}?key={key}`) — guests use the existing public schedule page on the main SPA, not the lab app
- **Dashboard integration preserved** — the `#/dashboard/scheduler/{id}` route still works; both paths are functional
