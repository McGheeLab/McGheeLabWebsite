# V3.35 — Scheduler Boolean (Yes/No) Questions

## Goal
Let a scheduler owner add custom boolean check-mark questions (e.g., "Are you available for a lab tour?") that each invited guest answers when they fill in their availability. Answers are visible to the owner in the admin view and surface on the confirmed-guests card in the public view.

## Approach
Extend the existing `guestFields` pattern (Talk Summary / Discussion Questions / Presentation Link) with a parallel list-typed field, `booleanQuestions`, that the owner edits as a dynamic list in the Schedule Settings panel.

### Data model
- `schedules/{id}.booleanQuestions: [{ id: string, text: string }]` — owner-defined questions. `id` is a stable uuid so answers survive text edits.
- `participants/{key}.booleanAnswers: { [questionId]: true }` — only stores checked answers (absent = unchecked). Keeps participant docs small and avoids stale entries after questions are deleted (old ids simply get ignored).

### UI surfaces
1. **Admin → Schedule Settings → Guest Fields** — new "Yes/No Questions" subsection with:
   - List of existing questions (text input + "Remove" button per row).
   - "+ Add Question" button appends a blank row.
   - Saved together with the rest of the settings form on "Save Settings".
2. **Guest view → Details card** — one checkbox per question, rendered only if the schedule defines any. Checked state reflects `speaker.booleanAnswers`. Saved via the existing "Save Details" button.
3. **Admin guest table** — new "Responses" column showing `N/M` yes answers when questions exist.
4. **Public / Confirmed speakers card** — if a guest answered yes to any question, render a small "Confirmed:" line listing those question texts.

### Files modified
- `scheduler.js` — core renderer/wire logic:
  - `setupFormHTML()` + `wireSetupForm()`: render + persist `booleanQuestions`.
  - `guestViewHTML()`: render checkboxes; adjust `hasAnyFields` so Details card appears when boolean questions exist.
  - `wireGuestForm()`: collect checkbox state into `booleanAnswers` on save.
  - `confirmedSpeakersHTML()`: render yes-answered question texts.
  - `adminViewHTML()`: add Responses column.
  - `defaultGuestInstructions()`: mention boolean questions if defined.
- `apps/scheduler/app.js` — seed `booleanQuestions: []` in `createFormHTML()` submit handler so newly-created schedules have the field present.
- `sw.js` — bump `CACHE_VERSION` from 3 → 4 per project convention.
- `CodeLog/Updates/CHANGELOG.md` — V3.35 Added entry.
- `CodeLog/Architecture/ARCHITECTURE.md` — note `booleanQuestions` alongside `guestFields` description; bump header version.

## Key decisions
- **Stable question `id`s** rather than positional indices. Editing a question's text after guests have answered keeps their "yes" intact; deleting a question drops only that answer.
- **Sparse `booleanAnswers` map** (only `true` entries) — avoids needing to reconcile the answer map every time the question list changes.
- **Admin renders inline**, not as its own settings subsection tab. Keeps parity with the existing `gf-*` toggles to stay discoverable.
- **No required/validation semantics** — questions are informational only; leaving them unchecked is always valid.
