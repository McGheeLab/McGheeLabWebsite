# V3.21 — Lab Meeting App

## Goal
Build out the Lab Meeting app (`apps/meetings/`) from its current stub into a fully functional tool for managing weekly lab meetings. The app handles presentation rotation scheduling, collaborative agenda building, shared meeting notes, action item tracking with cross-meeting accountability, and post-presentation cross-pollination signals.

## Approach
Follow the same vanilla JS IIFE + Firestore + auth-bridge pattern used by the Huddle app. Sidebar + main content layout with five sections: Next Meeting, Schedule, Archive, My Items, Settings.

## Key Decisions
- **Fixed rotation**: Admin sets presenter order, system auto-assigns for the semester
- **Embedded arrays**: Agenda items, action items, and feedback stored as arrays within the meeting document (lab meetings have <15 items each — well within Firestore limits)
- **Any member can add agenda items**: Low friction — encourages participation
- **Cross-pollination via reaction chips**: "Interesting", "Questions", "Collaborate", "Relevant" — lightweight signals that create conversation starters
- **Auto-carry**: Unchecked agenda items and open action items surface on the next meeting

## Data Model
- `meetings/{meetingId}` — One doc per meeting with date, presenter, agendaItems[], actionItems[], feedback[], notes
- `meetingConfig/settings` — Single config doc with rotation order, semester dates, skip dates, defaults

## Firestore Security Rules
- `meetings`: Authenticated read/create/update; admin-only delete
- `meetingConfig`: Authenticated read; admin-only write
