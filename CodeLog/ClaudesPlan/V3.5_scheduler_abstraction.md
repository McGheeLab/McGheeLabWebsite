# V3.5 — Scheduler Abstraction

## Goal
Extract the dual-mode scheduler (freeform + sessions) from class-scheduler.js into a standalone, reusable `scheduler.js` module. The scheduler can then be embedded anywhere on the site — class pages, standalone event pages, conference scheduling, etc.

## Two Modes

### Freeform (when2meet-style)
- Admin picks specific days and a time window with configurable increments (15/30/60 min)
- Grid: columns = days, rows = time blocks
- Speakers click+drag to paint their available times (green highlight)
- Admin sees aggregate heatmap: green gradient proportional to how many speakers are available per block
- No slot assignment — this mode is for availability polling

### Sessions
- Admin defines named time blocks (e.g., "Mon 8:30–10:30", "Mon 10:30–12:30")
- Speakers check off which sessions they can attend (preference voting)
- Admin runs greedy optimization to assign speakers to slots (bipartite matching)
- 1-to-1 assignment: each speaker gets one slot

## Architecture

### New file: `scheduler.js` (~650 lines)
Standalone IIFE exporting `window.McgheeLab.Scheduler`. Zero module state — all state passed via config object.

**Public API:**
```javascript
McgheeLab.Scheduler = {
  render(config),        // returns HTML string for the full scheduler UI
  wire(containerId, config),  // attaches listeners into the container

  // Sub-components (for granular embedding)
  renderGrid(config),
  renderConfirmedSpeakers(speakers, expanded, mode),

  // Pure logic
  expandSchedule(scheduleConfig),
  optimizeSchedule(speakers),

  // Utilities
  heatColor(count, max),
  fmtTime(timeString),
  slotLabel(slotId, allSlots)
};
```

**Config object:**
```javascript
{
  scheduleId: string,
  schedule: object,           // mode, dates, slotDefs, etc.
  speakers: array,
  currentSpeaker: object|null,
  viewType: 'admin'|'speaker'|'student',
  useKeyAuth: boolean,
  adminViewMode: 'admin'|'speaker'|'student',
  previewSpeakerIdx: number,
  // Callbacks (dependency injection)
  onSaveSpeaker: async (id, data) => {},
  onSaveSchedule: async (data) => {},
  onAddSpeaker: async (data) => {},
  onDeleteSpeaker: async (id) => {},
  onRefresh: async () => {},
  onSwitchView: (mode, speakerIdx) => {},
  buildInviteURL: (scheduleId, key) => string
}
```

### Modified file: `class-scheduler.js` (~600 lines)
Retains class builder only. Consumes scheduler via `McgheeLab.Scheduler.render(config)` / `.wire(containerId, config)`.

## Script Loading Order
```html
<script defer src="scheduler.js"></script>
<script defer src="class-scheduler.js"></script>
```

## Key Decisions
- **Zero module state in scheduler** — all state passed via config; the host (class builder or any other page) owns the state
- **Callback-based persistence** — scheduler never touches ScheduleDB directly; host provides save/delete callbacks
- **Shared helpers duplicated** — `esc()` copied into scheduler.js (3 lines); no shared utility file needed
- **Grid renderers return HTML** — pure functions; wire functions handle DOM insertion
- **CSS unchanged** — scheduler classes already namespaced (`.schedule-grid`, `.ff-cell`, etc.)
