# V2.4 — Video Upload in Stories

## Goal
Allow users to upload videos (mp4/webm) to story sections alongside or instead of images, with proper format validation, size limits, and correct rendering on public pages.

## Key Decisions

### Media Upload Zone
- Section media area accepts both images and videos
- File input `accept` changed to `image/*,video/mp4,video/webm`
- Upload zone label updated to "Click or drag an image or video here"
- File type detected on drop/select → routes to image pipeline or video pipeline

### Video Pipeline
- Accepted formats: `video/mp4`, `video/webm`
- Size limit: 50MB (videos are larger than images; Firebase Storage rule updated)
- No client-side transcoding — uploaded as-is to Firebase Storage
- Single file upload (no multi-resolution like images)
- Stored at `stories/{storyId}/section_{id}/video.{ext}`
- Download URL stored in section data as `video` field

### Data Model
- Each section gains optional `video` field (URL string) alongside existing `image` field
- A section can have text + image, text + video, or text only (not both image and video)
- `collectData()` includes `video` from `sectionVideos` map
- When a video is uploaded to a section that already has an image, the image is cleared (and vice versa)

### Rendering
- `buildStoryHTML()` in app.js checks for `block.video` and renders `<video>` tag with controls, playsinline, preload="metadata"
- Firestore story augmentation maps `sec.video` through to story blocks
- Preview modal renders `<video>` for sections with video data

### Storage Rules
- Story media rule updated: content type matches `image/.*` OR `video/mp4` OR `video/webm`
- Size limit increased to 50MB for stories path

## Files Modified
- `user-system.js` — sectionBlockHTML, handleMedia (new), wireSectionBlock, collectData, preview, restore
- `app.js` — buildStoryHTML, Firestore story augmentation section mapping
- `user-styles.css` — Video preview styles in editor and public pages
- `storage.rules` — Updated content type and size rules for stories
