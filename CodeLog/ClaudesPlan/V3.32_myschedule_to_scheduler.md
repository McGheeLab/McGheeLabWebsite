# V3.32 — Move My Schedule from Huddle to Scheduler App

## Goal
Move "My Schedule" from the Huddle into the Scheduler app as a full-featured personal calendar with layered data. The Huddle keeps read-only access for Team Availability and Rundown overlap detection.

## Approach

### Shared Modules
- Created `apps/shared/schedule-utils.js` — pure time/week utility functions (getWeekDays, fmtTime, timeToSlot, etc.) extracted from huddle, reusable by both apps.
- Created `apps/shared/schedule-service.js` — schedule data layer (CRUD, subscriptions, resolution logic) extracted from huddle. Manages Firestore subscriptions to huddleScheduleTemplates and huddleScheduleOverrides. Both apps load it.

### Scheduler App Enhancement
- Added two-tab layout: **My Schedule** (default) | **My Schedulers** (existing)
- My Schedule tab features:
  - **Layer panel** with toggles: Recurring (green), Overrides (blue/red), Calendar (purple), Custom (user color)
  - **Mode selector**: Recurring / Special Date / Blackout
  - **Weekly grid** (24h × 7 days) with zoom, drag-to-create, click-to-edit
  - **Calendar event layer** from CalendarService with dismiss (X) buttons
  - **Custom events** with title, color picker (12 presets), repeat toggle
  - **Block editor modals** for schedule blocks and custom events
  - **Week navigation** and "Copy Mon → Weekdays" utility

### Calendar Event Dismissal
- Added `dismissEvent(eventId)`, `restoreDismissed()`, `getDismissedCount()`, `isDismissed(eventId)` to CalendarService
- Dismissed event IDs persisted in `userSettings/{uid}.calendar.dismissedEventIds`
- `getEventsForDate()` auto-filters dismissed events
- "Restore All Dismissed" button in layer panel

### Huddle Refactor
- Removed ~500 lines: renderMySchedule, wireMySchedule, block modals, schedule CRUD/subscription/resolver functions, schedule state variables and constants
- Kept Team Availability tab — refactored to use `McgheeLab.ScheduleService.resolveScheduleForUser()` and `resolveAvailabilityOverlap()`
- Kept Rundown overlap detection — refactored to use shared service
- Bootstrap calls `ScheduleService.init()` instead of local subscription functions
- Week nav calls `ScheduleService.setWeekOffset()` instead of local re-subscribe

### Firestore
- New collection `scheduleCustomEvents` with auth read, owner CRUD, admin delete

## Key Decisions
- Schedule data layer is shared (both apps read the same Firestore collections)
- Scheduler app owns the write path; huddle is read-only consumer
- Local utility functions kept in huddle (used by plan grid) — will be migrated to ScheduleUtils in future refactor
- Calendar events rendered as a distinct visual layer, not just blackout injection
