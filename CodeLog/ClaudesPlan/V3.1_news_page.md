# V3.1 — News Page

## Goal
Add a new "News" page to the site where contributors, editors, and admins can create news posts about lab events, poster sessions, conferences, research highlights, new papers, lab outings, etc. Posts support images/videos, reactions, and comments — just like research stories.

## Approach
Reuse the existing patterns from the research stories system:
- **Firestore collection**: `newsPosts` (separate from `stories` which are research-specific)
- **Storage path**: `news/{postId}/` for media
- **Reactions & comments**: Reuse existing `reactions` and `comments` collections — comments/reactions already use a `storyId` field; we'll add news posts with the same field name for compatibility
- **Dashboard**: New "My News Posts" section alongside "My Stories"
- **Editor**: Simplified version of the story editor — no project/team/references (since news is freeform)
- **Public page**: `#/news` — feed of published news posts with reactions + comments

## Files to Modify
1. **user-system.js** — DB operations for news posts, news editor (render + wire), dashboard section
2. **app.js** — `renderNews()`, `buildNewsCard()`, `wireNewsFeedInteractions()`, router case
3. **index.html** — Add "News" nav link
4. **firestore.rules** — Rules for `newsPosts` collection
5. **storage.rules** — Rules for `news/` media path
6. **user-styles.css** — Minimal new styles (mostly reuse story-feed classes)
7. **CHANGELOG.md** — Document changes

## Key Decisions
- News posts do NOT require a project — they're freeform
- No team/mentor/contributor assignments — just an author
- Same reactions + comments system as research stories (reuse `loadReactions`/`loadComments`)
- News post editor is simpler than story editor: title, description, sections (text + optional image/video), category tag
- Category tags: "Event", "Conference", "Paper", "Highlight", "Lab Life", "Other"
- Any contributor/editor/admin can create news posts
- Publishing follows same flow: contributors submit for review, editors/admins publish directly
