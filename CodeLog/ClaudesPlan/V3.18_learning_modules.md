# V3.18 — Learning Modules

**Date:** 2026-04-09
**Goal:** Add a learning modules system that lets admins add any standalone HTML lesson page from a shared library to any class, view them within the SPA with auto-navigating headers, and manage linkages from the class builder UI.

## Approach

1. **Standalone HTML module pages** live in `modules/{folder}/` directories (e.g., `modules/bme576/`). Each is a fully self-contained `.html` file — zero modifications needed.
2. **Module manifest** — `modules/manifest.json` is an auto-generated index of all `.html` files organized by folder. Generated by `scripts/scan_modules.py` which extracts `<title>` tags for display names.
3. **SPA iframe viewer** — navigating to `#/classes/{classId}/modules/{filename}` renders a nav header bar (back, prev/next, homework) above an iframe. The iframe `src` is resolved from the module's `folder` field in Firestore.
4. **Add Module modal** — admin clicks "+ Add Module", gets a modal with title input + folder-organized dropdown + live search. Any module from any folder can be added to any class.
5. **Homework picker modal** — same search/browse pattern for linking uploaded assignment files.
6. **Data model:** `modules[]` array on existing `schedules/{classId}` Firestore doc. Each entry includes `folder` to locate the HTML file.

## Files Modified

- `class-builder.js` — `SECTION_REG` entry; `renderModulesBody()` admin card list + public link list; `openAddModuleModal()` with manifest fetch, dropdown, search; `openHomeworkPicker()` with file picker; `openFilePicker()` reusable grouped+searchable modal; `wireModuleEditors()`; `gatherContentFromDOM()`; `persistAll()` includes modules; `renderModuleViewer()` / `wireModuleViewer()`
- `app.js` — Routing for `#/classes/{classId}/modules/{filename}`
- `user-styles.css` — Module viewer nav bar, module admin cards, file picker modal, add module modal styles

## Files Created

- `scripts/scan_modules.py` — Scans `modules/` directories, extracts `<title>` tags, writes `manifest.json`
- `modules/manifest.json` — Auto-generated file index (run `scan_modules.py` after adding new HTML files)
- `modules/shared/module-header.css` — Standalone header styles (fallback for direct file access)
- `modules/shared/module-header.js` — Standalone Firestore-reading header (skips when in iframe)
- `modules/_template.html` — Starter template for new module pages
- `modules/bme295c/lesson-01.html` — Example module

## Key Decisions

- **Iframe approach** — Module HTML files stay 100% untouched. The SPA wraps them in an iframe with a native nav header above.
- **Cross-class modules** — Any HTML file from any folder can be added to any class. The `folder` field on each module entry lets the viewer find the file.
- **Manifest-based file discovery** — Since GitHub Pages can't list directories, `scan_modules.py` generates a JSON manifest that the admin UI fetches for dropdowns and search.
- Modules stored as array on the schedule doc — no new Firestore collections or security rules needed.
