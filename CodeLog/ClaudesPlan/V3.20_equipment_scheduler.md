# V3.20 — Equipment Scheduler Lab App

## Goal
Build a full equipment booking system as a lab app. Students schedule time on shared devices (microscopes, bioprinters, etc.), admins manage equipment and training requirements, and bookings sync to Google Calendar.

## Approach
- Implement as a standalone lab app at `apps/equipment/` following existing app patterns (auth bridge, embedded/standalone modes)
- Four Firestore collections: `equipment` (catalog), `equipmentBookings` (reservations), `equipmentTraining` (per-user certifications), `equipmentSettings` (global config)
- Real-time calendar via `onSnapshot()` listeners
- Google Calendar push-sync using Google Identity Services OAuth (admin-driven)

## Key Decisions
1. **Separate collections** (not subcollections) for bookings — enables cross-device date-range queries
2. **Client-side permission enforcement** — training/cert checks happen in browser; Firestore rules enforce auth + ownership
3. **Admin-held Google Calendar token** — only admin connects OAuth; regular users don't need Google accounts
4. **Soft-delete for cancellations** — `status: 'cancelled'` preserves audit trail
5. **Co-operator flow** — undergrads select a grad+ co-operator at booking time when required by device config

## Features
- Weekly + monthly calendar views with per-device filtering
- Priority color coding (normal/high/urgent/maintenance)
- Booking creation with conflict detection + duration constraints
- Training/certification requirement system
- Co-operator requirement for lower-level users
- Admin: equipment CRUD, training management, settings
- Google Calendar push-sync (create/update/cancel events)
- Real-time updates across all connected clients
