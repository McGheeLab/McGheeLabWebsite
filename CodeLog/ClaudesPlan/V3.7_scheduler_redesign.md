# V3.7 — Scheduler Redesign: Bare-Bones + Calendar Day Picker

## Goal
Simplify the scheduler from a feature-heavy tool to a clean, bare-bones scheduling experience. Three clear views (admin/guest/public). Session setup: define time slots first → pick days from a calendar → edit the resulting grid.

## Three Views

| View | Who | What they do |
|------|-----|-------------|
| **Admin** | Creator/owner | Set up sessions + days, manage guests, assign results |
| **Guest** | Invited person (via key URL) | Pick available slots, edit their response anytime |
| **Public** | Anyone with schedule link | See the finalized schedule cleanly |

## Session Setup Flow (Admin)

1. **Define session times** — Add standard time slots (e.g., 9:00–10:00, 10:30–12:00)
2. **Pick days from calendar** — Month-view calendar with multi-select (click to toggle days)
3. **Review grid** — Days (columns) × session times (rows), editable table format

### Data Model Change
```
// OLD: date range generates all days
{ startDate: '2026-06-08', endDate: '2026-06-12', slotDefs: [...] }

// NEW: explicit selected days
{ selectedDays: ['2026-06-08', '2026-06-10', '2026-06-12'], slotDefs: [...] }

// Backward compat: if selectedDays absent, fall back to startDate/endDate range
```

## View Name Mapping
- `speaker` → `guest` (accept both internally)
- `student` → `public` (accept both internally)
- `admin` stays `admin`

## Simplified Admin View
1. Grid (days × sessions) with availability heatmap
2. Guest management (add name/email → get invite link, assign slots)
3. Settings (collapsed by default)
- Optimization appears as a button only when there are unassigned guests

## Files Modified
- `scheduler.js` — Full redesign: calendar component, session-first setup, view renaming, simplified layouts
- `user-system.js` — Update viewType references (speaker→guest)
- `class-scheduler.js` — Update viewType references
- `app.js` — No changes needed (routes don't reference view types)
