# V3.6 — Dashboard Schedulers (Private Key-Only Access)

## Goal
Let any logged-in user create and manage scheduler tasks from their dashboard. Scheduler pages are private — accessible only via an invite key URL given to the invited person.

## Architecture

### User Flow
1. **Dashboard → "My Schedulers" card** — User sees list of their schedulers, can create new ones
2. **Create scheduler** — Simple form: title, mode (sessions/freeform), date range
3. **Manage scheduler** — Navigate to `#/dashboard/scheduler/{id}` for admin view (add speakers, get invite URLs, configure schedule, optimize)
4. **Speaker access** — Invite URL: `#/schedule/{id}?key={uuid}` — renders speaker view (no login required)

### Routes
- `#/dashboard` — Shows "My Schedulers" card with create + list
- `#/dashboard/scheduler/{id}` — Admin/owner view of a scheduler (full scheduler engine)
- `#/schedule/{id}?key={uuid}` — Private speaker access (key-only, no auth required, no public link)

### Data Model
Reuses existing Firestore collections:
- `schedules/{id}` — Schedule metadata (title, mode, dates, ownerUid, etc.)
- `participants/{key}` — Speaker records with invite keys

New field on schedule docs:
- `ownerUid` — (already exists) set to creating user's UID

### Implementation

1. **user-system.js** — Dashboard:
   - Add "My Schedulers" card to `renderDashboard()`
   - Add `refreshSchedulerList()` — queries schedules where `ownerUid == user.uid`
   - Add create form handler — creates schedule via `ScheduleDB.saveSchedule()`
   - Add delete button — deletes schedule + all participants

2. **user-system.js** — Scheduler management page:
   - `renderSchedulerEditor(id)` — returns HTML shell with back link + container
   - `wireSchedulerEditor(id)` — loads schedule + speakers, renders scheduler in admin view using `McgheeLab.Scheduler.render/wire`

3. **user-system.js** — Private schedule page:
   - `renderSchedulePage(id)` — returns HTML shell for speaker/key access
   - `wireSchedulePage(id)` — extracts key from URL, loads speaker, renders scheduler in speaker view

4. **app.js** — Routes:
   - Add `#/schedule/{id}` case (private key-only access)
   - Add `#/dashboard/scheduler/{id}` sub-route

### Key Decisions
- Reuses `McgheeLab.Scheduler` engine and `ScheduleDB` — no new scheduler code
- Private route (`#/schedule/{id}`) separate from class routes (`#/classes/{id}`)
- No nav links to `#/schedule/` — only accessible via invite URL
- Any logged-in user (not just admin) can create schedulers
- Invite URL uses `#/schedule/{id}?key=` (not `#/classes/`)
