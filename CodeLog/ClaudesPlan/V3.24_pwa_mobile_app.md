# V3.24 — Progressive Web App (PWA) & Push Notifications

## Goal
Make the lab apps installable as mobile applications on iPhone and Android while keeping them hosted on the website. A single codebase that serves both web and mobile — no app store required.

## Approach
Convert the existing SPA into a Progressive Web App (PWA) by adding:
1. **Web App Manifest** — enables "Add to Home Screen" with app icon, splash screen, standalone mode
2. **Service Worker** — caches shell + app files for fast loading and offline capability
3. **Push Notifications** — Firebase Cloud Messaging for real-time alerts (chat, meetings, bookings)
4. **Install Prompt UI** — custom banners guiding users to install on Android/iOS
5. **Standalone Mode Polish** — CSS adjustments for display without browser chrome

## Key Decisions
- `start_url: "/#/apps"` — lab members want quick access to tools, not the public site
- `display: standalone` — removes browser chrome for native app feel
- Two separate service workers: `sw.js` (caching) and `firebase-messaging-sw.js` (push)
- Notification permission requested contextually (after using chat/equipment), not on page load
- Push tokens stored in Firestore subcollection `users/{uid}/pushTokens`

## Files Created
- `manifest.json` — Web app manifest
- `sw.js` — Caching service worker
- `firebase-messaging-sw.js` — FCM push notification service worker
- `push-notifications.js` — Client-side push permission/token module
- `favicon.ico` — Root favicon
- `icons/` — App icons at 13 sizes (72–512px, maskable, apple-touch-icon, favicons)

## Files Modified
- `index.html` — PWA meta tags, iOS tags, favicons, FCM SDK, SW registration
- `app.js` — Standalone detection, install prompt, SW update toast, push init
- `styles.css` — Install banner, notification prompt, toast, standalone mode CSS
- `lab-apps.js` — Install banner + notification prompt on apps hub page
- `apps/shared/auth-bridge.js` — Offline-aware auth wall fallback
- `firestore.rules` — `pushTokens` subcollection rules
- `apps/*/index.html` (8 files) — theme-color, apple-touch-icon meta tags

## Platform Notes
- **Android Chrome**: Full PWA support — install prompt, standalone, push notifications
- **iOS Safari**: "Add to Home Screen" via Share button. Push requires iOS 16.4+ and PWA must be installed. No badge counts.
- **Push backend**: VAPID key from Firebase Console required. Start with admin-triggered notifications, then add Cloud Functions for event-driven (chat messages, meeting reminders)

## Future: Phase 2 (Capacitor)
If App Store / Play Store presence is needed, Capacitor can wrap the PWA in native shells. The PWA foundation built here is a prerequisite for that path.
