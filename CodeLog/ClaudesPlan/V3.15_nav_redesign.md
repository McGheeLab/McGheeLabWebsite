# V3.15 — Grouped Navigation (Desktop Dropdowns + Mobile Tab Bar)

## Goal
Replace the flat hamburger nav with grouped, platform-optimized navigation:
- **Desktop (≥768px):** Horizontal nav with dropdown groups on hover/click
- **Mobile (<768px):** Fixed bottom tab bar with group sheets
- **Mission** is now the splash page — accessed via logo click, not a nav link

## Navigation Groups
| Group | Pages | Desktop | Mobile |
|---|---|---|---|
| Research | Research, Projects, News | Dropdown | Bottom sheet |
| People | Team, Opportunities | Dropdown | Bottom sheet |
| Classes | Classes | Direct link | Direct tab link |
| Other | Lab Apps, Dashboard, Admin, Login | Inline (auth-gated) | "More" bottom sheet |

## Desktop Implementation
- `#desktop-nav` contains `<li class="nav-group">` items with `<button class="nav-group-btn">` + `<ul class="nav-dropdown">`
- Hover opens dropdown, mouseleave closes it. Click toggles for touch-screen desktops.
- Click outside or route change closes all dropdowns.
- Group button gets `.group-active` class when any child route is current.
- Chevron SVG inside button rotates on open.

## Mobile Implementation
- 4 bottom tabs: Research (group), People (group), Classes (direct), More (group)
- Group tabs (`button.tab-group-btn[data-group]`) toggle a `#group-sheet-{name}` bottom sheet
- Each sheet has backdrop dismiss, link-click dismiss, Escape key dismiss
- Only one sheet open at a time (opening one closes others)
- `NAV_GROUPS` map in JS drives group-aware active highlighting

## Files Modified
- `index.html` — Restructured `#desktop-nav` with dropdown groups, new `#bottom-tabs` with group buttons, 3 group sheets (research, people, more)
- `styles.css` — Section 03b: `.nav-group`, `.nav-group-btn`, `.nav-dropdown` dropdown styles. Section 03c: `.group-sheet` bottom sheet styles (replaced `.more-sheet`)
- `app.js` — `setupDesktopDropdowns()`/`closeAllDropdowns()` for desktop, `setupBottomTabs()`/`toggleGroupSheet()`/`closeAllSheets()` for mobile, group-aware `setActiveTopNav()` with `NAV_GROUPS` map
- `user-system.js` — No changes needed (existing IDs still work)
