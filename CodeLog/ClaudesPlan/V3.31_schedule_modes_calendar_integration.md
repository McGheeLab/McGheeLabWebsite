# V3.31 — Schedule Modes & Calendar Integration

## Goal
1. Add color-coded scheduling modes to the Huddle "My Schedule" so users can distinguish recurring, special-date, and blackout blocks.
2. Move calendar import from the Activity Tracker to the Settings page, make it persistent, and share it across apps.
3. Calendar events auto-inject as blackout blocks in the Huddle schedule.

## Approach

### Schedule Modes
- Added `mode` field to schedule blocks: `'recurring'` (green), `'special'` (blue), `'blackout'` (red).
- **Green (recurring):** Saved to `huddleScheduleTemplates` — repeats every week.
- **Blue (special date):** Saved as `huddleScheduleOverrides` with `action: 'add'` — one-time availability.
- **Red (blackout):** Saved as `huddleScheduleOverrides` with `action: 'add'` — non-recurring unavailable time.
- Mode selector UI: three pill-shaped toggle buttons above the schedule grid.
- Modal adapts per mode: green shows type dropdown, blue pre-fills available, red pre-fills unavailable with reason/rigidity.
- Block colors on grid now use `MODE_COLORS` instead of reason colors / user colors.
- Backward compat: existing blocks without `mode` inferred from source/type.

### Shared Calendar Service
- Created `apps/shared/calendar-service.js` — extracted and refactored calendar code from activity tracker.
- Exposes `McgheeLab.CalendarService` global with `init()`, `getEventsForDate()`, `connectGoogle()`, `connectOutlook()`, `importICSFile()`, `fetchAll()`, `onChange()`, etc.
- Config persisted in Firestore `userSettings/{uid}.calendar`.
- OAuth tokens remain session-lived (sessionStorage with shared keys).
- ICS URLs auto-fetched on load. Auto-refresh on configurable interval.

### Settings Calendar UI
- New "Calendar Integration" section in Settings app between Notifications and App Administration.
- Provider cards: Google Calendar, Outlook/Microsoft 365, Apple Calendar (ICS).
- Auto-refresh interval dropdown (15m / 30m / 1h / 2h / 4h / disabled).
- "Sync to Huddle" toggle — auto-block calendar events as blackout times.

### Activity Tracker Migration
- Removed ~620 lines of calendar code from activity tracker.
- Removed "Calendar" sidebar tab.
- Daily view "unlogged events" prompt now sources from `McgheeLab.CalendarService.getEventsForDate()`.

### Calendar → Huddle Integration
- `resolveScheduleForUser()` injects calendar events as blackout blocks for the current user when `huddle.autoBlock` is enabled.
- Calendar blocks are read-only (clicking shows toast instead of edit modal).
- Added `parseCalTimeToHHMM()` utility for time format conversion.

## Key Decisions
- Mode determines storage location (template vs override) — no "save to template" checkbox needed.
- Calendar tokens stay in sessionStorage (expire hourly anyway); only config/URLs persist in Firestore.
- Calendar blocks only injected for the current user in Huddle (private data).
