# V3.13 — Repeatable, Collapsible, Nameable Sections

## Goal
Refactor class builder sections so they behave as repeatable building blocks rather than singleton types. Each section instance gets a custom name, can be collapsed, and carries a "type" for future styling hooks.

## Current Behavior
- `SECTION_REG` defines fixed section types (overview, syllabus, simulations, etc.)
- Each type can only be added once (uniqueness enforced across all tabs)
- Section labels come from `SECTION_REG` and can't be customized
- Text content stored globally on `_classData[field]`, shared by type
- No collapse/expand capability

## New Behavior
1. **Repeatable**: Any section type can be added multiple times
2. **Nameable**: Each section instance has a custom name (prompted on creation, editable via double-click)
3. **Collapsible**: Each section has a collapse/expand toggle (persisted)
4. **Type-styled**: Each section gets a CSS class `cb-section-type-{key}` for future styling (e.g., simulation sections with light blue borders)

## Data Model Change
Section object: `{ key, id, name, collapsed, content, storagePath, widgets }`
- `key`: section type from SECTION_REG (determines component + styling)
- `id`: unique block ID (unchanged)
- `name`: custom display name (new)
- `collapsed`: boolean (new)
- `content`: per-instance text content for text-component sections (new — replaces `_classData[field]`)
- `storagePath`: per-instance file storage path for file-component sections (new)
- `widgets`: array of widget objects (unchanged)

## Key Changes

### Remove Uniqueness
- `addSection()` no longer checks `getAllUsedSections()`
- `buildSectionDropdownItems()` no longer disables used types
- Prompt for section name on creation

### Per-Instance Content
- Text sections store content in `section.content` instead of `_classData[reg.field]`
- File sections use `section.storagePath` (set to `section.id` for new, legacy path for migrated)
- `gatherContentFromDOM()` writes to section objects
- `persistAll()` no longer copies `_classData` fields — tabs carry everything

### Collapse
- Toggle button in section chrome (both admin and readonly)
- `cb-collapsed` CSS class hides section body
- State persisted on section object

### Migration
- `migrateLegacy()` populates `name`, `collapsed`, `content`, `storagePath` from legacy data
- Existing text content migrated from `schedule[reg.field]` to `section.content`
- Existing file paths preserved via `storagePath = reg.path || key`

## Files Modified
- `class-builder.js` — all section logic
- `styles.css` — collapse styles, section type borders
