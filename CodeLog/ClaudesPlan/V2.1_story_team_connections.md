# V2.1 — Story-Team Connections

## Goal
Connect stories to team members (author, contributors, mentor) and add a reference system (publications, patents, presentations, posters). Display team and references in expanded story views, and show reference badge links on unopened cards.

## Approach

### Data Model
Stories gain two new top-level fields:
- `team`: `{ author: {uid, name, photo}, contributors: [{uid, name, photo}], mentor: {uid, name, photo} | null }`
- `references`: `{ publications: [{title, url, detail}], patents: [...], presentations: [...], posters: [...] }`

Both are optional — existing stories without these fields render exactly as before (graceful degradation).

### Display Layer (app.js)
1. **`normalizeTopicOrProject()`** — Extended to pass through `team` and `references` from content.json or Firestore
2. **New helpers** after `buildStoryHTML()`:
   - `buildStoryTeamHTML(team)` — Renders a team grid with photo, name, role
   - `buildStoryRefsHTML(refs)` — Renders categorized reference lists
   - `buildRefLinksHTML(refs)` — Compact badge links for unopened cards
   - `buildExpandedHTML(storyBlocks, data)` — Composes team + sections + refs
3. **Render functions** (`renderResearch`, `renderProjects`, Firestore augmentation) — Switch from `buildStoryHTML` to `buildExpandedHTML` for expanded content; add `buildRefLinksHTML` to card faces

### Editor (user-system.js)
- **Team fields**: Author (read-only, current user), Mentor (dropdown of all users), Contributors (multi-select with add button + chip display)
- **Reference inputs**: Repeatable rows with type dropdown, title, URL, detail; add/remove buttons
- **`collectData()`** — Extended to build `team` and `references` objects from form state

### Styles (user-styles.css)
- `.story-team`, `.story-team-grid`, `.story-team-member` — Responsive flex grid for team display
- `.story-refs`, `.story-refs-category` — Categorized reference lists
- `.ref-badges`, `.ref-badge` — Compact colored badges for card faces
- `.team-fields`, `.contributor-chips`, `.ref-block-row` — Editor form styles
- Mobile breakpoint at 600px for stacked layouts

## Files Modified
- `app.js` — normalizer + 4 new helpers + 3 render function updates
- `user-system.js` — renderStoryEditor + wireStoryEditor + collectData + refBlockHTML
- `user-styles.css` — ~200 lines of new styles
- `CodeLog/Updates/CHANGELOG.md` — V2.1 entry

## Key Decisions
- Team data is denormalized (name/photo stored in story document) to avoid extra reads on public pages
- References grouped by type (object with arrays) rather than flat array — cleaner rendering
- Ref badges link to the first item of each category — simple heuristic for card-level quick links
- All new features are additive — zero regressions for stories without team/refs data
