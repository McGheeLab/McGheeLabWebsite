Background: Experimental Data and Measurement Noise

What is experimental data?

In any experiment, we measure a state variable (the quantity we observe, such as volume, fluorescence intensity, or cell count) at a set of conditions or time points. The true relationship between the state variable and the conditions is governed by parameters — quantities like growth rates, diffusion coefficients, or binding affinities that we cannot measure directly but want to infer.

Forward problem vs. inverse problem

The forward problem is easy: given known parameter values, compute what the data should look like. For example, if a cell population doubles every 24 hours starting from 1000 cells, the population at time \(t\) is \(N(t)=1000\cdot 2^{t/24}\). Plug in parameters, get predictions.

The inverse problem is hard: given noisy measurements, recover the unknown parameters. This is the subject of the entire module. We observe scattered data points and ask: "Which parameter values would produce a model curve closest to these measurements?"

Types of measurement noise

  • Additive noise: \(y_{\text{obs}} = y_{\text{true}} + \varepsilon\), where \(\varepsilon\sim\mathcal{N}(0,\sigma^2)\). The error magnitude is constant regardless of the signal size.
  • Multiplicative noise: \(y_{\text{obs}} = y_{\text{true}}\cdot(1+\varepsilon)\). Error grows proportionally with the signal — common in biological measurements where larger volumes have larger absolute errors.
  • Heteroscedastic noise: the noise variance changes across the measurement range. For tumor spheroids, volume is estimated from diameter via \(V=\frac{\pi}{6}d^3\); small errors in \(d\) get cubed, so large spheroids have much larger volume uncertainty.

Suppose the true value at some time point is \(y_{\text{true}}=10.0\) and the noise level is \(\sigma=0.08\) (8% multiplicative).

An observed value might be \(y_{\text{obs}} = 10.0\times(1 + 0.08\times z)\) where \(z\sim\mathcal{N}(0,1)\).

If \(z=1.2\), then \(y_{\text{obs}} = 10.0\times 1.096 = 10.96\). If \(z=-0.7\), then \(y_{\text{obs}}=10.0\times 0.944=9.44\).

With 5 replicates, you might get: \(10.96,\; 9.44,\; 10.32,\; 9.87,\; 10.61\). The scatter around the true value is the noise we must account for during fitting.

Background: Least Squares Curve Fitting

What is least squares?

Each data point \(i\) has a residual — the vertical gap between the observed value \(y_i\) and the model prediction \(\hat{y}_i = f(x_i,\boldsymbol{\theta})\):

\[ r_i = y_i - f(x_i,\,\boldsymbol{\theta}) \]

For \(y = mx + b\) with data \((1, 3.1),\; (2, 4.8),\; (3, 7.2)\) and current guess \(m=2, b=1\):

Predictions: \(\hat{y}_1 = 2(1)+1 = 3\), \(\hat{y}_2 = 2(2)+1 = 5\), \(\hat{y}_3 = 2(3)+1 = 7\)

Residuals: \(r_1 = 3.1 - 3 = 0.1\), \(r_2 = 4.8 - 5 = -0.2\), \(r_3 = 7.2 - 7 = 0.2\)

Positive means the model underpredicts; negative means it overpredicts. We collect all \(n\) gaps into a residual vector \(\mathbf{r}=[r_1,r_2,\ldots,r_n]^T\).

The residual sum of squares (RSS)

We square each gap (so positive and negative errors do not cancel) and add them all up:

\[ \text{RSS}(\boldsymbol{\theta}) = \sum_{i=1}^{n}r_i^2 = \mathbf{r}^T\mathbf{r} \]

Continuing: RSS \(= 0.1^2 + (-0.2)^2 + 0.2^2 = 0.01 + 0.04 + 0.04 = 0.09\)

This is a small RSS, suggesting \(m=2, b=1\) is close to the best fit.

The expression \(\mathbf{r}^T\mathbf{r}\) is the dot product of the residual vector with itself — the squared length of the vector. Fitting = minimizing this length.

The Jacobian — sensitivity of predictions to parameters

Before the optimizer can decide which parameter to change, it needs to know: "If I nudge parameter \(\theta_j\) by a tiny amount, how much does each prediction move?" That table of sensitivities is the Jacobian matrix \(J\):

\[ J_{ij} = \frac{\partial f(x_i,\,\boldsymbol{\theta})}{\partial \theta_j} \]

For \(f(x) = mx + b\): \(\frac{\partial f}{\partial m} = x\) and \(\frac{\partial f}{\partial b} = 1\)

At our 3 data points: \(J = \begin{bmatrix} 1 & 1 \\ 2 & 1 \\ 3 & 1 \end{bmatrix}\)

Interpretation: changing \(b\) shifts all predictions by the same amount (column 2 = all 1s). Changing \(m\) affects later points more (column 1 = 1, 2, 3).

Row \(i\) = data point. Column \(j\) = parameter. Entry = "how sensitive is prediction \(i\) to parameter \(j\)?"

Gauss-Newton parameter update

Given the current residuals \(\mathbf{r}\) and the sensitivity table \(J\), the best local correction is:

\[ \Delta\boldsymbol{\theta} = \underbrace{(J^TJ)^{-1}}_{\text{scaling}} \underbrace{J^T\mathbf{r}}_{\text{direction}} \]

Computing the update:

\(J^T J = \begin{bmatrix}1&2&3\\1&1&1\end{bmatrix}\begin{bmatrix}1&1\\2&1\\3&1\end{bmatrix} = \begin{bmatrix}14&6\\6&3\end{bmatrix}\)

\(J^T \mathbf{r} = \begin{bmatrix}1&2&3\\1&1&1\end{bmatrix}\begin{bmatrix}0.1\\-0.2\\0.2\end{bmatrix} = \begin{bmatrix}0.3\\0.1\end{bmatrix}\)

\((J^TJ)^{-1} = \frac{1}{14\cdot3-36}\begin{bmatrix}3&-6\\-6&14\end{bmatrix} = \frac{1}{6}\begin{bmatrix}3&-6\\-6&14\end{bmatrix}\)

\(\Delta\theta = \frac{1}{6}\begin{bmatrix}3&-6\\-6&14\end{bmatrix}\begin{bmatrix}0.3\\0.1\end{bmatrix} = \frac{1}{6}\begin{bmatrix}0.3\\0.2\end{bmatrix} = \begin{bmatrix}0.05\\0.033\end{bmatrix}\)

Update: \(m_{\text{new}} = 2 + 0.05 = 2.05\), \(b_{\text{new}} = 1 + 0.033 = 1.033\)

\(J^T\mathbf{r}\) asks "which parameter direction would most reduce the current gaps?" and \((J^TJ)^{-1}\) scales that direction so we do not overshoot.

Standard errors from the fit

At convergence, the same \(J^TJ\) tells us parameter uncertainty:

\[ \text{Cov}(\hat{\boldsymbol{\theta}}) \approx s^2(J^TJ)^{-1}, \quad s^2 = \frac{\text{RSS}}{n-p} \]

Standard errors: With RSS = 0.09, \(n=3\), \(p=2\): \(s^2 = 0.09/(3-2) = 0.09\)

\(\text{Cov} = 0.09 \times \frac{1}{6}\begin{bmatrix}3&-6\\-6&14\end{bmatrix} = \begin{bmatrix}0.045&-0.09\\-0.09&0.21\end{bmatrix}\)

SE(\(m\)) = \(\sqrt{0.045} = 0.212\), SE(\(b\)) = \(\sqrt{0.21} = 0.458\)

So \(m = 2.05 \pm 0.21\) and \(b = 1.03 \pm 0.46\). The intercept is less precisely determined than the slope.

where \(n\) = data points and \(p\) = number of parameters. Standard errors are the square roots of the diagonal: \(\text{SE}(\hat{\theta}_j) = \sqrt{\text{Cov}_{jj}}\).

1-parameter fit: Fit \(y=a\cdot x\) to \((1,2.1),\;(2,3.9),\;(3,6.2)\).

RSS = \((2.1-a)^2+(3.9-2a)^2+(6.2-3a)^2\). Setting \(\frac{d\text{RSS}}{da}=0\):

\(\hat{a}=\frac{\sum x_i y_i}{\sum x_i^2}=\frac{2.1+7.8+18.6}{1+4+9}=\frac{28.5}{14}\approx 2.036\)

Background: Residual Analysis

What are residuals?

After fitting, each data point has a residual: \(e_i = y_i - \hat{y}_i\). These residuals are the "leftover" variation that the model did not capture. If the model is correct and the noise is well-behaved, residuals should look like random scatter around zero.

Three diagnostic patterns

  • Random scatter (good): No systematic pattern. Points bounce above and below zero with no trend. This means the model captures the data structure adequately.
  • U-shape or curvature (bad): Residuals are negative at the edges and positive in the middle (or vice versa). This means the model shape is wrong — it is missing curvature that the data contains.
  • Fan / trumpet shape (heteroscedastic): Residuals spread out as the predicted value increases. This means the noise variance is not constant — larger predictions have larger errors. Weighted least squares or a log-transform may help.

Goodness-of-fit statistics

Step 1 — Residual sum of squares:

\[ SS_{\text{res}} = \sum_{i=1}^{n}(y_i - \hat{y}_i)^2 \]

Step 2 — Total sum of squares (how much total variation exists before any model):

\[ SS_{\text{tot}} = \sum_{i=1}^{n}(y_i - \bar{y})^2, \quad \bar{y}=\frac{1}{n}\sum y_i \]

Step 3 — Coefficient of determination:

\[ R^2 = 1 - \frac{SS_{\text{res}}}{SS_{\text{tot}}} \]

\(R^2=0.95\) means the model captures 95% of the data's spread. If the model is no better than predicting the mean, \(R^2=0\). For very bad models, \(R^2\) can be negative.

RMSE — average residual magnitude in original units:

\[ \text{RMSE} = \sqrt{\frac{SS_{\text{res}}}{n}} \]

Key distinction: \(R^2\) is relative (fraction of variance), RMSE is absolute (units of the data). A model can have \(R^2=0.99\) and still be systematically wrong if residuals show patterns.

Observed \(y=[2,\;4,\;7]\), predicted \(\hat{y}=[1.8,\;4.1,\;6.5]\).

Residuals: \(e=[0.2,\;-0.1,\;0.5]\). RSS = \(0.04+0.01+0.25=0.30\).

Mean \(\bar{y}=4.33\). \(SS_{\text{tot}}=5.44+0.11+7.11=12.66\).

\(R^2=1-0.30/12.66=0.976\). RMSE = \(\sqrt{0.30/3}\approx 0.316\).

Background: R² Pitfalls and Linearizing Transforms

Why R² can mislead

\(R^2\) measures the fraction of variance explained by a linear fit. But a high \(R^2\) from a linear fit does not mean the linear model is correct — it just means the data has a trend. Consider Anscombe's quartet: four wildly different datasets that all yield the same \(R^2\approx 0.67\) and the same linear regression line. The residual plot distinguishes them; the number alone does not.

For nonlinear models, \(R^2\) defined as \(1-SS_{\text{res}}/SS_{\text{tot}}\) can exceed 1 or be negative, and it no longer has the clean "fraction of variance explained" interpretation. Always check residuals alongside any summary statistic.

Linearizing transforms

When the true relationship is nonlinear, a variable transformation can convert it into a form where linear regression (and \(R^2\)) is valid:

  • Log-linear for exponentials: \(y=ae^{bx}\) becomes \(\ln y = \ln a + bx\) — linear in \(\ln y\) vs \(x\).
  • Log-log for power laws: \(y=ax^b\) becomes \(\ln y = \ln a + b\ln x\) — linear in \(\ln y\) vs \(\ln x\).
  • Reciprocal for Michaelis-Menten: \(y=V_{\max}x/(K_m+x)\) becomes \(1/y = (K_m/V_{\max})(1/x)+1/V_{\max}\) — the Lineweaver-Burk plot.

Data: \(x=[1,2,3,4,5]\), \(y=[2.7,\;7.4,\;20.1,\;54.6,\;148.4]\). These follow \(y=e^x\).

Linear fit gives \(R^2=0.82\). But \(\ln y=[0.99,\;2.00,\;3.00,\;4.00,\;5.00]\) gives a perfect linear fit with \(R^2=1.00\). The log-linear transform reveals the true structure.

Background: Model Comparison with AIC and BIC

Why better fit does not mean better model

A model with more parameters will always fit the data at least as well as a simpler one. But extra parameters can overfit — they memorize noise rather than capturing the real pattern. On new data, the overfit model predicts poorly. We need a principled way to balance goodness of fit against model complexity.

Information criteria — Occam's razor made quantitative

Define the ingredients:

  • \(n\) = number of data points
  • \(k\) = number of estimated parameters
  • \(\text{RSS}\) = residual sum of squares from fitting

Akaike Information Criterion (AIC):

\[ \text{AIC} = \underbrace{n\ln\!\left(\frac{\text{RSS}}{n}\right)}_{\text{fit quality (lower = better)}} + \underbrace{2k}_{\text{complexity penalty}} \]

The first term rewards good fit. The second penalizes each additional parameter by exactly 2 units.

Bayesian Information Criterion (BIC):

\[ \text{BIC} = \underbrace{n\ln\!\left(\frac{\text{RSS}}{n}\right)}_{\text{same fit term}} + \underbrace{k\ln(n)}_{\text{stronger penalty}} \]

Since \(\ln(n)>2\) whenever \(n>8\), BIC penalizes extra parameters more harshly than AIC.

ΔAIC interpretation

  • \(\Delta\text{AIC}=0\text{-}2\): Models are essentially indistinguishable
  • \(\Delta\text{AIC}=2\text{-}7\): Some evidence favoring the lower-AIC model
  • \(\Delta\text{AIC}>10\): Strong evidence against the higher-AIC model

Model A: RSS=50, \(k=2\), \(n=60\). Model B: RSS=45, \(k=4\), \(n=60\).

AIC\(_A=60\ln(50/60)+4=60(-0.182)+4=-6.95\)

AIC\(_B=60\ln(45/60)+8=60(-0.288)+8=-9.25\)

\(\Delta\text{AIC}=2.30\). Model B is slightly preferred, but the evidence is modest.

Background: Parameter Identifiability

Structural vs. practical identifiability

  • Structural identifiability: Can we EVER identify these parameters, even with perfect (noise-free, infinite) data? This is a property of the model equations themselves.
  • Practical identifiability: Can we identify them with THIS data (finite, noisy, limited range)? This depends on the experimental design.

From covariance to standard error (SE)

Recall from Tab 2 that after fitting, we obtain the parameter covariance matrix \(\text{Cov}(\hat{\boldsymbol{\theta}}) \approx s^2(J^TJ)^{-1}\). This is a matrix where:

  • Diagonal entries \(\text{Cov}_{jj}\) are the variances of individual parameter estimates — they measure how much each parameter would scatter if we repeated the experiment many times.
  • Off-diagonal entries \(\text{Cov}_{jk}\) (where \(j \neq k\)) are covariances between pairs — they measure whether parameters tend to move together.

The Standard Error (SE) of a parameter is the square root of its variance — it converts the abstract variance back into the same units as the parameter itself:

\[ \text{SE}(\hat{\theta}_j) = \sqrt{\text{Cov}(\hat{\boldsymbol{\theta}})_{jj}} \]

For example, if \(\text{Cov}_{11} = 0.010\), then \(\text{SE}(\hat{\theta}_1) = \sqrt{0.010} = 0.100\). This means the parameter estimate is uncertain by about ±0.1. A 95% confidence interval is approximately \(\hat{\theta}_j \pm 1.96 \cdot \text{SE}(\hat{\theta}_j)\).

Small SE = well-determined parameter. Large SE = uncertain parameter. If SE is comparable to the parameter value itself, the estimate is essentially meaningless.

The correlation matrix — detecting parameter tradeoffs

Covariance values depend on the scale of each parameter, making them hard to compare directly. To get a unitless measure of how strongly two parameters are linked, we normalize each covariance by the standard errors of both parameters:

\[ r_{jk} = \frac{\text{Cov}_{jk}}{\text{SE}_j \cdot \text{SE}_k} = \frac{\text{Cov}_{jk}}{\sqrt{\text{Cov}_{jj} \cdot \text{Cov}_{kk}}} \]

This gives the correlation coefficient \(r_{jk}\in[-1,+1]\):

  • \(r_{jk}=0\): parameters are uncorrelated — good
  • \(|r_{jk}|\) near 1: parameters trade off strongly — one can compensate for the other
  • Danger threshold: \(|r_{jk}|>0.9\) usually indicates practical non-identifiability

Profile likelihood — a global identifiability test

The 5-step algorithm:

  1. Fix parameter \(\theta_j\) at a specific value (away from the best fit)
  2. Re-optimize all other parameters to get the best possible fit given that \(\theta_j\) is locked
  3. Record the resulting RSS
  4. Repeat for a grid of \(\theta_j\) values
  5. Plot RSS vs. \(\theta_j\). Sharp parabola = identifiable. Flat curve = not identifiable

Given: \(\text{Cov}=\begin{bmatrix}0.010&0.018\\0.018&0.040\end{bmatrix}\) for parameters \((\alpha,\beta)\).

Step 1 — Standard errors (square roots of diagonal entries):

SE(\(\alpha\)) = \(\sqrt{\text{Cov}_{11}} = \sqrt{0.010} = 0.100\)     SE(\(\beta\)) = \(\sqrt{\text{Cov}_{22}} = \sqrt{0.040} = 0.200\)

Interpretation: \(\alpha\) is known to ±0.10, \(\beta\) to ±0.20 (roughly).

Step 2 — Correlation (off-diagonal normalized by both SEs):

\(r_{\alpha\beta} = \frac{\text{Cov}_{12}}{\text{SE}(\alpha) \cdot \text{SE}(\beta)} = \frac{0.018}{0.100 \times 0.200} = \frac{0.018}{0.020} = 0.90\)

Verdict: At \(r = 0.90\), the parameters strongly trade off — increasing \(\alpha\) can be compensated by increasing \(\beta\). The confidence region is a long tilted ellipse, not a compact blob. This is a warning sign for identifiability.

Understanding the Covariance Matrix — Step by Step

The covariance matrix is a 2×2 table (for 2 parameters) that encodes everything about parameter uncertainty. This section builds up what each entry means, then shows how the matrix creates the confidence ellipse. Drag the sliders to see each concept in action.

What the matrix looks like for two parameters α and β:

Cov = [ Var(α)    Cov(α,β) ]
       [ Cov(α,β)   Var(β)     ]

Reading the diagonal (top-left and bottom-right):

  • Var(α) = SE(α)² — the variance of α. Larger variance = more uncertainty in α. The standard error SE is just √Var.
  • Var(β) = SE(β)² — same for β.

Reading the off-diagonal (top-right and bottom-left, always equal):

  • Cov(α,β) = r · SE(α) · SE(β) — the covariance. It tells you whether the two parameters move together (positive) or in opposite directions (negative).
  • Normalizing: r = Cov(α,β) / (SE(α) · SE(β)) gives the correlation coefficient r ∈ [-1, +1].

What the matrix tells you at a glance: Large diagonal = uncertain parameters. Large off-diagonal relative to diagonal = parameters that trade off against each other. If |r| > 0.9, the parameters are practically non-identifiable as a pair.
